# Java 后端架构师题库大模型审稿记录

## 审稿范围

- 目录：`questions/java-architect`
- 数量：200 道题
- 目标：检查题目重复、表达模板化、可记忆性、面试现场感和是否需要合并/替换。

## 主要发现

1. **题目覆盖完整，但部分题意接近**
   - `Spring Cloud 服务注册、发现与配置治理` 与 `Dubbo、Spring Cloud 与服务治理取舍` 边界偏近。
   - `Kubernetes 上 Java 服务的资源治理` 与 `Kubernetes HPA/VPA 与 Java 容量规划` 边界偏近。
   - `AI Agent 工具调用、状态与审计` 与 `Agent 工具权限与人工确认机制` 边界偏近。

2. **内容存在模板化重复**
   - 多题复用“主链路保持简单可靠”“旁路观测”“值班负责人”等表达。
   - 优点是结构稳定，缺点是连续阅读时疲劳，面试现场感不足。

3. **记忆锚点还可以更像真人讲题**
   - 原有内容偏架构文档，更适合精读。
   - 面试复习需要更短的记忆口诀、领域类比和 60 秒口述版本。

## 已执行修改

1. **合并/替换近义题**
   - 将 `Dubbo、Spring Cloud 与服务治理取舍` 调整为 `Dubbo 到 Spring Cloud Alibaba 的迁移治理`。
   - 将 `Kubernetes HPA/VPA 与 Java 容量规划` 调整为 `KEDA 事件驱动伸缩与队列消费扩容`。
   - 将 `Agent 工具权限与人工确认机制` 调整为 `Agent 人在回路审批与风险分级设计`。

2. **新增领域拟人化表达**
   - JVM：急诊科医生
   - Java 并发：交通调度员
   - Spring Boot：应用装配师
   - Spring Cloud：城际交通枢纽调度员
   - MySQL：账房先生
   - Redis：前台接待
   - Kafka：物流分拣中心负责人
   - 高可用：灾备演练指挥官
   - 架构设计：总设计师
   - RAG 工程：图书馆参考咨询员
   - Agent 架构/工程化：带实习生的值班组长
   - 模型服务：云上调度经理

3. **优化记忆和面试表达**
   - 保留每题的“场景、边界、链路、风险、验证”记忆骨架。
   - 将拟人化理解、60 秒回答和转场话术改为按领域变化。
   - 让候选人在复习时能从“读材料”切换到“开口答题”。

## 后续建议

- 后续如果继续扩题，优先补充“真实事故案例”和“代码/配置片段”，不要再单纯扩数量。
- AI 相关题建议定期按行业变化刷新，如 Agent 权限、eval、RAG 安全、模型成本治理。
- 如果要进一步降低重复感，可以把生成器改成按题型生成不同答案模板，而不是单一长模板。
