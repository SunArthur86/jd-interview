# 【拼多多 AI 中台】LLM 多模态应用（图文/语音）怎么落地？

> JD 依据："图像/语音/NLP 模型服务、大模型多模态"。

## 一、多模态 LLM 架构

```
图像 ──→ 视觉编码器（ViT）──┐
                              │
文本 ──────────────────────── →│
                              │ 投影层（MLP/Q-Former）
                              ↓
                          LLM（如 Qwen2/Llama）
                              ↓
                          多模态输出（文本/可控）
```

**关键组件**：
- 视觉编码器：ViT/CLIP 编码图像为向量序列
- 投影层：把视觉向量对齐到 LLM 的 embedding 空间（MLP/Q-Former/交叉注意力）
- LLM：处理统一的多模态 token 序列

## 二、主流 VLM

| 模型 | 特点 | 适用 |
|------|------|------|
| **GPT-4V/4o** | OpenAI 闭源，能力最强 | 通用（API） |
| **Qwen-VL** | 阿里开源，中文好 | 国内业务（推荐） |
| **LLaVA** | 学术开源 | 研究/定制 |
| **InternVL** | 上海 AI Lab | 中文场景 |
| **CogVLM** | 智谱 | 多任务 |
| **Gemini** | Google 多模态原生 | 通用 |

## 三、训练流程

```
阶段 1：模态对齐预训练
  - 数据：海量图文对（如 LAION）
  - 目标：让视觉编码器和 LLM embedding 空间对齐
  - 冻结 LLM，只训练投影层

阶段 2：多模态预训练
  - 数据：高质量图文指令
  - 全参数训练

阶段 3：SFT（指令微调）
  - 数据：多模态对话（VQA/OCR/推理）
  - 提升指令遵循

阶段 4：RLHF（可选）
  - 人类偏好对齐
```

## 四、典型应用场景

### 1. 商品图理解（电商核心）
```
输入：商品图片 + "提取商品属性"
输出：{"品牌": "X", "类目": "手机", "颜色": "黑色", "价格区间": "..."}

应用：
  - 商品自动打标（省人工）
  - 商品搜索（图搜图/图文搜索）
  - 推荐召回（视觉相似）
  - 详情页生成（图+文）
```

### 2. 语音搜索
```
用户语音 → ASR（Whisper）→ 文本 → 搜索/推荐 → 结果 → TTS（VITS）→ 语音回复

技术栈：
  - ASR：Whisper（OpenAI）/Paraformer（阿里）
  - LLM：意图理解 + 对话
  - TTS：VITS/ChatTTS
```

### 3. 视频内容审核
```
输入：用户上传视频
处理：
  - 抽帧（每秒 1 帧）
  - VLM 识别每帧（违规/敏感）
  - 时序聚合（连续违规判定）
输出：违规标记 + 时间戳

替代传统 CV 模型，更通用（一个 VLM 替代多个分类器）
```

### 4. 图文生成
```
输入：商品标题 + "生成详情图描述"
输出：图文并茂的详情描述

或：
输入：图片 + "生成营销文案"
输出：贴合图片的文案
```

### 5. 智能客服（图文）
```
用户："这个商品 [图] 怎么用？"
VLM 识别图 → 商品类型 → 知识库召回使用说明 → LLM 生成回复
```

## 五、工程化挑战

### 1. 推理性能
```
VLM 比纯文本慢：
  - 视觉编码（ViT）开销
  - 图像 token 占用上下文（一张图 = 几百 token）

优化：
  - 视觉编码器量化（INT8）
  - 动态分辨率（按需裁剪）
  - 视觉 token 压缩（如 TokenPacker）
  - KV Cache 复用
```

### 2. 数据成本
```
图文对数据贵：
  - 标注成本（人工标注图文关系）
  - 合成数据（用 GPT-4V 生成）
  - 业务数据回流（用户上传图文）
```

### 3. 评估
```
VLM 评估复杂：
  - 通用 benchmark（VQA/MME/MMBench）
  - 业务自定义（如商品属性准确率）
  - 人工评估（必要）
```

### 4. 多模态 RAG
```
传统 RAG：文本向量化检索
多模态 RAG：
  - 图文统一向量化（CLIP）
  - 跨模态检索（用文本搜图/用图搜文）
  - 多模态结果给 VLM 生成
```

## 六、拼多多实战

### 案例 1：商品自动打标
```
痛点：每天新增百万商品，人工打标跟不上
方案：
  - VLM 提取商品属性（品牌/类目/颜色/材质/风格）
  - 准确率 90%+，人工只审核高价值类目
  - 节省 80% 人工

工程：
  - Qwen-VL-72B 部署（vLLM）
  - 批量推理（夜间跑）
  - 入特征仓供搜索/推荐用
```

### 案例 2：语音搜索
```
用户语音"找便宜的耳机" →
  - Whisper ASR → 文本
  - LLM 意图理解 → 搜索 query
  - 推荐召回 → 结果
  - VITS TTS → 语音回复
端到端延迟 < 3s
```

### 案例 3：内容审核
```
传统：训练多个 CV 分类器（暴力/色情/广告/...）
VLM：
  - 一个 VLM 替代所有分类器
  - 提示词定义违规类型
  - 新违规类型加 prompt 即可（不用重训）
  - 准确率持平或更高
```

## 七、技术选型

```
通用 VLM：Qwen-VL-Max（API）或 Qwen-VL-72B（自部署）
轻量 VLM：Qwen-VL-7B/LLaVA-7B（端侧/边缘）
ASR：Whisper-Large / Paraformer
TTS：VITS / ChatTTS / CosyVoice（阿里）
向量检索（多模态 RAG）：CLIP 向量 + Milvus

推理引擎：vLLM（支持 VLM）/ TensorRT-LLM
量化：AWQ INT4（VLM 也能量化）
```

## 八、未来方向

- **原生多模态**（如 GPT-4o/Gemini）：端到端训练，非拼接
- **视频理解**：长视频时序建模
- **多模态 Agent**：VLM + Function Calling（看图操作）
- **多模态生成**：Sora/Wenze（图生视频）
- **端侧多模态**：手机/AR 眼镜

## 九、底层本质

多模态 LLM 本质是**"用统一表示空间处理图/文/音/视频"**——视觉编码器把图像变向量，投影到 LLM 空间，LLM 处理统一 token 序列。让 AI 能"看"和"听"，覆盖电商核心场景（商品图/语音搜索/视频审核/图文生成）。工程上要平衡精度/成本/延迟，量化 + batching + 动态分辨率是关键优化。

## 常见考点

1. **VLM 怎么处理高分辨率图**？——动态分辨率（按需切片）/滑窗/分层编码，权衡精度和 token 占用。
2. **CLIP 是什么**？——对比学习图文对齐模型，图像和文本编码到同一空间，用于多模态检索/分类。
3. **多模态和单模态 RAG 区别**？——单模态只文本向量化检索；多模态用 CLIP 类模型统一编码，支持跨模态检索（文搜图/图搜文）。
