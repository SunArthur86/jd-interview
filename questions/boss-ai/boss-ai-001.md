# 【巨剧核 AI 陪伴】Agent 编排全链路怎么设计？

> JD 依据："负责 Agent 编排，包括角色人设、剧情上下文、用户输入、用户记忆、工具调用、模型路由、输出审核、结果返回等链路。"

## 一、Agent 编排的本质

裸调 LLM API：
```
user_input → LLM → response
（无角色、无记忆、无约束）
```

Agent 编排（AI 陪伴场景）：
```
user_input
   ↓
[1] 上下文组装：人设 + 剧情上下文 + 用户记忆 + 当前输入
   ↓
[2] 模型路由：按场景/成本/延迟选模型
   ↓
[3] 工具调用（如需）：LLM 决定调搜索/数据库/API
   ↓
[4] 生成：LLM 生成回复
   ↓
[5] 输出审核：安全 + 合规 + 人设一致性
   ↓
[6] 返回用户 + 更新记忆/状态
```

## 二、上下文组装（prompt 工程）

```
system: 你是「小雨」，一个温柔的陪伴角色。人设：{personality}。当前剧情：{story_context}。
user_history_summary: 用户叫小明，喜欢科幻，最近情绪低落。
recent_dialog: [最近 5 轮对话]
user: {current_input}
```

**关键工程**：
- **人设系统**：角色性格/语气/边界，存 DB，动态注入
- **剧情上下文**：当前故事线进度（短剧/互动剧情）
- **记忆摘要**：长期记忆压缩（不能全塞，见记忆系统题）
- **recent dialog**：近 N 轮原文（短期上下文）

## 三、模型路由

```python
def route_model(scene, complexity, budget):
    if scene == "simple_chat" and budget == "low":
        return "qwen-turbo"           # 便宜快
    if complexity == "high" or scene == "emotional_support":
        return "claude-sonnet"        # 强推理/共情
    return "gpt-4o-mini"              # 默认均衡
```

**路由维度**：
- 场景（闲聊/情感支持/工具任务）
- 复杂度（简单/复杂推理）
- 成本/延迟预算
- **fallback 链**：主模型挂 → 备模型 → 兜底小模型

## 四、工具调用（Tool Calling）

LLM 决定调外部工具增强能力：
```
用户："帮我查下今天的天气"
LLM 输出：{tool: "weather_api", args: {city: "上海"}}
工具执行 → 结果回填 → LLM 生成最终回复
```

AI 陪伴的工具：搜剧情设定、查用户画像、生成图片、触发音视频任务。

## 五、输出审核（三层）

```
生成内容
   ↓
[安全审核] 关键词/分类模型（暴力/色情/自伤）
   ↓
[合规审核] 隐私泄露/广告/政治敏感
   ↓
[人设一致性] 是否符合角色语气/边界（小模型判别）
   ↓
通过 → 返回；不通过 → 重生成/降级回复
```

**流式审核**（不拖慢响应）：边生成边审，命中风险即截断。

## 六、底层本质

Agent 编排本质是**"把无状态 LLM 升级为有状态、有约束、有能力的工程系统"**。每一环（上下文/路由/工具/审核）都是对裸 LLM 的"工程化补丁"，让通用模型变成可信赖的产品服务。

**这是"模型能力"到"产品服务"的鸿沟跨越**——90% 的 AI 产品工作量在编排工程，而非模型本身。

## 常见考点
1. **为什么用 LangGraph 而不是 LangChain**？——LangGraph 图式编排（节点+边），适合多步/循环/条件分支；LangChain 链式偏线性。
2. **工具调用怎么保证安全**？——工具白名单 + 参数校验 + 权限控制 + 审计日志。
3. **Agent 怎么处理长对话**？——记忆系统（短期原文 + 长期摘要 + 向量检索），见记忆系统题。


## 结构化回答

**30 秒电梯演讲：** 聊到Agent 编排全链路怎么设计，我的理解是——Agent 编排是把"用户输入+人设+上下文+记忆"组装成 prompt，经模型路由选模型、工具调用增强、输出审核过滤，最终返回安全且符合人设的回复——本质是一条可编排的决策流水线。打个比方，Agent 编排像导演拍一场戏——演员（人设）拿到剧本（上下文）和道具（工具），按场景（用户输入）表演，最后过审（输出审核）才能上映。

**展开框架：**
1. **编排链路** — 人设+上下文+记忆+输入 → prompt → 模型路由 → 工具调用 → 输出审核 → 返回
2. **模型路由** — 按场景/成本/延迟选模型（小模型兜底、大模型复杂推理）
3. **工具调用（Tool Calling）** — 让 LLM 调用外部能力（搜索/数据库/API）
