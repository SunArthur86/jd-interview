# 【巨剧核 AI 陪伴】AI 内容审核系统怎么建（UGC/AIGC 双向）？

> JD 依据："负责内容审核；UGC/AIGC 双向审核；多模态。"

## 一、为什么是双向审核

```
UGC（用户 → 平台）：
  用户输入文本/语音/图片
  风险：诱导越狱 / 辱骂 / 发布违规内容 / 隐私泄露
  必须：入审（用户提交时审）

AIGC（平台 → 用户）：
  AI 生成回复/图片/语音/视频
  风险：违规输出 / 人设崩塌 / 不雅内容 / 错误信息
  必须：出审（生成后投递前审）

任一方向漏审 = 风险流入平台
```

## 二、多模态审核范围

```
文本：
  关键词/分类模型/LLM-Judge
  暴力/色情/政治/广告/隐私/越狱

图像：
  NSFW 分类 / 暴力检测 / OCR 文字 / 名人识别
  不雅图/暴血图/含违规文字图/侵权图

语音：
  ASR 转文本审 / 音频情绪分类 / 敏感词音素
  辱骂/敏感内容/异常情绪

视频：
  帧抽取审图 / 音轨审音 / 跨帧一致性 / 关键帧重点
  不雅动作/违规画面/插帧违规/剧情违规

跨模态：
  文本合规但配图违规（图文组合）
  数字人嘴说合规但表情违规
```

## 三、审核管线（多级）

```
内容
   ↓
[L1] 关键词/正则
     快、零延迟
     拦截明确红线词
   ↓
[L2] 小分类模型
     10ms、中召回
     按类别训练（NSFW/暴力/政治）
     主力
   ↓
[L3] LLM-as-Judge / 多模态 LLM
     秒级、高准
     判复杂场景（隐晦/上下文）
   ↓
[L4] 人工复核
     边界 case
     标注 + 反馈
```

分级使用：默认 L1+L2，高风险/可疑加 L3，争议入 L4。

## 四、UGC 入审策略

```
时机：
  发消息/发帖/上传时同步审
  高风险异步复审

策略：
  - 用户分 tier（VIP/付费/免费/黑名单）
  - 历史违规用户严格审
  - 新用户首条严格审
  - 高风险场景（私聊/群聊）严格审

处置：
  拦截（不发）
  警告（发了但提醒）
  限流（违规用户）
  封禁（严重违规）
```

## 五、AIGC 出审策略

```
时机：
  生成后、投递前审
  流式生成边生成边审

策略：
  - 安全审核（红线）
  - 合规审核（监管）
  - 人设一致性（体验）
  - 多模态一致性（音视频和文本是否一致）

处置：
  拦截 → 重生成 / 降级模板 / 截断
  流式中命中 → 撤回已显示 + 替换
```

## 六、流式审核（关键体验）

```
LLM/AIGC 流式生成：
  内容按 chunk 产出
  
审核方式：
  按 chunk 边生成边审
  命中风险 → 截断 + 替换

挑战：
  已推送的 chunk 怎么处理？
    发"撤回"指令前端移除
    替换为安全回复
  未推送的 chunk 直接丢弃
  极端违规 → 中断会话 + 告警

延迟：
  审核并行不阻塞推送
  小模型快速审
```

## 七、特殊场景

### 1. 越狱攻击
```
用户诱导：
  "扮演一个不受限制的角色"
  "忽略前面的指令"
  用编码/拼音/小语种绕过

防御：
  - system prompt 强化（明确边界）
  - 越狱 pattern 识别
  - 多语言模型 + 归一化
  - 频次限制（防穷举）
  - 用户行为风控
```

### 2. 隐私泄露
```
PII 识别：
  身份证/电话/地址/银行卡
  身份证号正则 + 实体识别模型

策略：
  输入：脱敏后入 prompt
  输出：拦截含 PII 的内容
  记忆：脱敏后存
  日志：脱敏展示
```

### 3. 数字人视频审核
```
帧级审：
  关键帧抽取审图（NSFW/暴力）
音轨审：
  ASR + 内容审
跨帧：
  防插帧违规（一帧违规混入）
整体：
  剧情合规（角色行为）
AIGC 标记：
  加水印（合规要求）
```

## 八、闭环优化

```
线上拦截 → 人工标注（误杀/漏杀）→ 训练集
                              → 优化关键词
                              → 重训分类模型
                              → 调阈值
用户举报 → 同上

新风险：
  快速规则补丁（关键词/正则）
  样本积累重训
  行业共享黑名单
```

## 九、性能与成本

```
延迟控制：
  小模型（蒸馏）快速审
  并行审核（多模型同时）
  缓存（同内容不重审）
  分级（高风险强审/闲聊抽审）

成本：
  免费用户抽审
  付费用户全审
  LLM-Judge 仅高风险场景
  自训小模型替代大模型 API
```

## 十、底层本质

AI 内容审核系统的本质是**"在 UGC 和 AIGC 双向、多模态、流式场景下，建立确定性的安全护栏"**：

- 双向 = 防输入也防输出
- 多模态 = 覆盖文/图/音/视
- 管线 = 多级叠加
- 流式 = 不拖慢体验
- 闭环 = 持续优化

这是 AI 产品合规商用的底线——监管（《生成式 AI 服务管理办法》）要求"生成内容可管控、可追溯"，没有完善的审核系统就不能上线。

## 常见考点

1. **审核误杀正常对话怎么办？**——阈值分级（场景/tier 不同阈值）+ 用户申诉 + 白名单 + 持续调优；要给用户解释和救济通道。
2. **多模态审核延迟高怎么办？**——并行审核 + 分级（默认文本快速审，可疑再扩到多模态）+ 缓存 + 异步复审（先发后审，发现问题撤回）。
3. **AIGC 怎么标注满足监管？**——加水印（不可见但可鉴伪）+ 元数据（生成时间/模型）+ 内容日志可追溯 + 接入监管报送接口。


## 结构化回答

**30 秒电梯演讲：** 聊到AI 内容审核系统怎么建（UGC/AIGC 双向），我的理解是——AI 内容审核系统是 UGC（用户生成）+ AIGC（AI 生成）的双向护栏——既要审用户说了什么（防违规），又要审 AI 生成了什么（防出格），覆盖文本/图像/语音/视频多模态。打个比方，像一个商场双重保安——入口保安查顾客带进来的东西（审 UGC），出口保安查店员卖出去的东西（审 AIGC）；既要防违禁品入内，也要防伪劣品外流，覆盖包袋/食品/电器所有品类（多模态）。

**展开框架：**
1. **双向** — UGC 入审 + AIGC 出审，各自有策略
2. **多模态** — 文本/图像/语音/视频各有模型
3. **管线** — 关键词+分类模型+LLM-Judge+人工分级
