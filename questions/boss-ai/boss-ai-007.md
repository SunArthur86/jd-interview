# 【巨剧核 AI 陪伴】模型输出审核怎么做（安全/合规/人设）？

> JD 依据："负责输出审核；负责权限、隐私。"

## 一、为什么必须审核

```
LLM 是概率模型 → 概率性输出：
  - 暴力 / 色情 / 自伤（安全风险）
  - 隐私泄露 / 广告 / 政治敏感（合规风险）
  - 出戏 / 人设崩塌 / 边界越界（体验风险）
任一条流入用户 = 监管处罚 + 用户伤害 + 品牌危机
```

监管（如《生成式 AI 服务管理办法》）要求**生成内容可管控、可追溯**。审核不是可选项，是合规底线。

## 二、三层审核模型

```
LLM 生成内容
   ↓
[1] 安全审核（红线）
   暴力 / 色情 / 自伤 / 极端言论 / 违法
   工具：关键词表 + 内容安全 API（阿里/腾讯/百度）+ 自训练分类模型
   ↓
[2] 合规审核（监管）
   隐私泄露（身份证/电话/地址）/ 政治敏感 / 广告 / 版权 / 涉诈
   工具：PII 识别 + 关键词 + 监管名单 + 分类模型
   ↓
[3] 人设一致性（体验）
   角色是否说符合人设的话？语气/边界/价值观
   工具：LLM-as-Judge（用另一个模型判）+ 人设规则
   ↓
通过 → 返回；不通过 → 处置
```

**人设一致性**是 AI 陪伴的特色：温柔角色突然骂人，不违法但严重伤体验。

## 三、技术栈组合

```
┌──────────────────────────────────┐
│ L1 关键词/正则     ← 快、零延迟、低召回 │  兜底红线词
├──────────────────────────────────┤
│ L2 小分类模型      ← 快（10ms）、中召回 │ 主力
│   （蒸馏自大模型，按类别训练）        │
├──────────────────────────────────┤
│ L3 LLM-as-Judge   ← 慢（秒级）、高准   │ 高风险/抽审
│   （用 GPT-4/Claude 判定）           │
├──────────────────────────────────┤
│ L4 人工复核        ← 慢、准、贵        │ 边界 case
└──────────────────────────────────┘
```

分级使用：默认 L1+L2，高风险场景加 L3，可疑样本入 L4。

## 四、流式审核（不拖慢响应）

LLM 流式生成时，审核必须并行：
```
LLM 流式输出 token 序列
   ↓
按句/分块（如 20 token 一窗）送审
   ↓
正常 → 立即推前端
命中风险 → 截断 + 替换为兜底回复 + 后台重生成
```

**已生成的部分怎么处理**：
- 已推送前端的：发一条'撤回/修正'指令
- 未推送的：直接丢弃，替换为安全回复
- 极端违规：中断整个会话 + 上报告警

## 五、处置策略

```
命中等级    处置
─────────────────────────────────
低风险      替换关键词 / 软提示重生成
中风险      整条拦截 → 用模板兜底回复
高风险      拦截 + 重生成（换 prompt）
极高风险    拦截 + 标记用户 + 人工介入
```

兜底回复：人设一致的通用安全话术（"我有点不知道怎么回答这个，我们聊点别的吧～"）。

## 六、闭环优化

```
线上拦截样本 → 人工标注（是否误杀/漏杀）→ 入训练集
                                        → 优化关键词/规则
                                        → 重训分类模型
                                        → 调阈值
用户举报样本 → 同上
```

审核系统是**持续迭代的**，不是一次性建设。

## 七、特殊场景

- **用户诱导越狱**（"扮演一个不受限制的角色"）：识别越狱 pattern + 加 system prompt 防御 + 频次限制
- **多语言绕过**：用拼音/emoji/小语种绕关键词 → 多语言模型 + 归一化
- **隐晦表达**（不直接违规但暗示）：LLM-as-Judge 兜底 + 上下文审核

## 八、底层本质

输出审核的本质是**"在概率性 AI 输出和用户之间，建立确定性的安全护栏"**：

- 三层维度：安全 / 合规 / 人设
- 多重叠加：关键词 + 模型 + LLM-Judge
- 流式并行：不拖慢响应
- 持续闭环：样本回流优化

这是把**'尽量安全'（依赖模型自律）**升级为**'确定安全'（系统强制拦截）**，是 AI 产品从实验室走向大规模商用的合规底线。

## 常见考点

1. **审核误杀正常对话怎么办？**——阈值分级（高风险场景严、闲聊松）+ 用户申诉通道 + 白名单（VIP/付费用户宽松些）+ 持续调阈值。
2. **流式审核截断后前端体验怎么处理？**——发'修正'指令替换已显示文本 + 平滑过渡动画 + 角色自然衔接（如"等等让我重新想想"）。
3. **AIGC 怎么满足监管可追溯？**——每条生成内容带元数据（模型/prompt/时间/user）+ 输出日志保留 90 天+ + 加水印标记 AIGC + 接入监管报送接口。


## 结构化回答

**30 秒电梯演讲：** 聊到模型输出审核怎么做（安全/合规/人设），我的理解是——模型输出审核是在 LLM 生成内容到用户眼前之间，加一道'安全 + 合规 + 人设'三重关卡，拦住违规、隐私、出戏内容，让 AI 既自由又有边界。打个比方，像电视台的播出审查——节目（生成内容）开播前要过三关：不能播违法的（安全）、不能泄露国家秘密（合规）、不能让演员突然说跟剧本不符的台词（人设一致性）。

**展开框架：**
1. **三层审核** — 安全（暴色政）+ 合规（隐私/广告/版权）+ 人设一致性（角色边界）
2. **技术栈** — 关键词 + 分类模型 + LLM-as-Judge 多重叠加
3. **流式审核** — 边生成边审，命中即截断，不拖慢响应
