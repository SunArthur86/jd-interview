# 【巨剧核 AI 陪伴】数据埋点与实验平台怎么做？

> JD 依据："负责数据埋点和实验。"

## 一、为什么要埋点和实验

```
拍脑袋决策：PM 觉得好 → 上线 → 留存跌了 → 不知道为什么
数据决策：假设 → 实验 → 数据验证 → 确定更好才全量
```

产品迭代本质是判断题，**感觉会骗人，只有数据不会**。

## 二、埋点事件模型

每个事件 5 要素：
```
{
  who:    user_id / device_id / 角色会话 id
  what:   event_name（如 message_sent / role_reply / thumb_up）
  when:   timestamp
  where:  页面/场景/来源（app_version, channel, scene）
  value:  参数（消息长度、角色 id、token 消耗、质量分）
}
```

**分层埋点**：
- 基础：启动/活跃/页面路径（标准分析）
- 业务：发消息/点赞/付费/邀请（产品核心）
- AI 特色：角色互动轮数/会话时长/LLM 延迟/token 成本/质量分/审核拦截

## 三、实验平台架构

```
┌──────────────────────────────────────┐
│ 实验配置中心                            │
│   实验 A：流量 50/50，指标=留存         │
│   实验 B：流量 30/70，指标=时长         │
└──────────────────────────────────────┘
            ↓ 分桶
┌──────────────────────────────────────┐
│ 流量分桶（按 user_id hash 正交分层）     │
│   每个用户落到各实验的具体桶里            │
└──────────────────────────────────────┘
            ↓
┌──────────────────────────────────────┐
│ 业务代码读取分桶 → 走对应逻辑           │
│ 埋点上报：用户命中的所有实验版本         │
└──────────────────────────────────────┘
            ↓
┌──────────────────────────────────────┐
│ 数据仓库 → 指标计算 → 显著性检验 → 报表  │
└──────────────────────────────────────┘
```

## 四、流量分桶与正交

```
单实验：user_id % 100 → 0-49 桶 A，50-99 桶 B

多实验并行（分层正交）：
  层 1（UI 实验）：user_id hash → 桶 A/B
  层 2（prompt 实验）：(user_id + 层2盐) hash → 桶 C/D
  层间正交 → 同一用户在不同实验独立分桶，不互相污染
```

这是 Google/Bolt 通用做法，支持同时跑几十上百个实验。

## 五、指标体系

```
北极星：用户长期价值（如月活 / 月会话数）
  ↓
核心指标：次日留存 / 7日留存 / 人均会话时长
  ↓
拆解指标：发消息率 / 角色互动轮数 / 付费转化
  ↓
护栏指标（防副作用）：投诉率 / 审核拦截率 / 成本/单次 / 崩溃率
```

**护栏指标**是关键——一个好实验不能是'时长涨了但投诉也涨了'。

## 六、AI 产品的特色指标

| 维度 | 指标 | 怎么采 |
|---|---|---|
| 质量 | LLM-as-Judge 评分 / 人工评分 | 抽样 + 自动 |
| 体验 | 单会话轮数 / 用户主动开启率 | 埋点 |
| 人设 | 人设一致性分（LLM 判） | 抽样 |
| 性能 | 首 token 延迟 / 流式 P95 | 监控 |
| 成本 | 单会话 token / 单次成本 | 计费 |

## 七、显著性检验

```
假设：A 比 B 留存高 3%
检验：
  样本量：每桶 1 万用户
  t 检验 → p < 0.05 → 差异显著（不是噪声）
  置信区间：[1.2%, 4.8%] → 真实差异范围

陷阱：
  - 样本不够 → 噪声当结论
  - 多次比较 → 假阳性（要 Bonferroni 校正）
  - 辛普森悖论 → 分桶不均导致整体结论反转
  - Peek 问题 → 中途偷看下结论（要序贯检验）
```

## 八、底层本质

实验平台的本质是**"把产品决策从经验主义升级为实证主义"**：

- 埋点 = 数据采集（事实）
- 实验 = 控制变量（方法）
- 显著性 = 推断可靠性（统计）

这是把'产品迭代'变成'科学实验'，让每个改动有据可依、可复现、可累积——是数据驱动公司的核心基础设施（字节、Meta、Google 都靠这套）。

## 常见考点

1. **小流量实验效果很好，全量后没效果？**——可能是霍桑效应（被关注就表现好）/ 样本偏差（早期用户偏）/ 流量分布差异；要做分群下钻分析。
2. **指标冲突（留存涨但成本也涨）怎么决策？**——看 ROI（留存收益 vs 成本）+ 战略导向（早期容忍成本换规模，成熟期控成本）+ 护栏指标硬约束。
3. **实验平台怎么不拖慢业务？**——SDK 轻量化（异步上报）+ 配置中心缓存（毫秒级读取）+ 实验配置热更（不发版）。
