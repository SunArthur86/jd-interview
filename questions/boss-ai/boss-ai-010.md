# 【巨剧核 AI 陪伴】服务稳定性与安全怎么做？

> JD 依据："负责稳定性：限流、降级、缓存、队列、日志、监控、报警、权限、隐私。"

## 一、为什么稳定性是底线

```
角色爆火 → 10 万人涌入 → LLM 网关限流 → 大量超时 → 用户骂街 → 媒体发酵
依赖模型厂商挂 → 服务全瘫 → 用户流失
单次会话成本失控 → 月底账单爆炸 → 公司亏钱
用户隐私泄漏 → 监管处罚 + 信任崩塌
```

稳定性是产品的**生命线**，安全是**合规底线**。

## 二、限流（防超载）

```
多级限流：
  网关层：IP / user_id 维度，防恶意刷
  应用层：接口级（发消息/生成视频分别限）
  模型层：LLM 网关 QPS / token/s 限流（保护 API 配额）

算法：
  令牌桶（允许突发）—— 主流
  漏桶（严格匀速）
  滑动窗口（精确计数）

分 tier：
  VIP：100 req/min
  付费：30 req/min
  免费：10 req/min
  + 全局兜底（总 QPS 不超 GPU 容量）
```

## 三、降级（保核心）

```
策略：核心保命，非核心关停

LLM 降级链：
  强模型 → 弱模型 → 兜底模板回复
  （高峰自动切弱模型，省钱省 GPU）

功能降级：
  视频/图片生成暂停（成本高）
  长上下文压缩（省 token）
  主动触发消息关闭（省调用）

触发条件：
  GPU 利用率 > 80%
  LLM 错误率 > 5%
  成本告警（超日预算）
```

## 四、缓存（减重复调用）

```
LLM 结果缓存：
  key = hash(persona_id + user_id + input + params)
  value = LLM 回复
  TTL = 短（30min，防过期信息）
  场景：FAQ 型问题（"你是谁"）命中率极高

语义缓存（进阶）：
  query embedding 相似度 > 0.95 → 命中
  风险：要严格过滤（避免错误命中）

热点缓存：
  热门角色人设 / 剧情进度 / 用户画像 → Redis
```

## 五、队列削峰（异步化）

```
实时请求：聊天主链路（用户在等）
异步任务：视频生成/批量推送/记忆重算 → 队列（见音视频任务题）

削峰：
  流量突增 → 入队等待 → worker 按 GPU 容量消费
  用户感知：进度提示 + 预计时间
```

## 六、监控告警（秒级发现）

**黄金四指标**：
- 延迟（P50/P95/P99 首 token 延迟）
- 流量（QPS / 并发会话）
- 错误（LLM 错误率 / 审核拦截率 / 5xx）
- 饱和度（GPU 利用率 / 队列堆积 / token 消耗速率）

**AI 特色指标**：
- 单会话成本 / token / 转化
- 模型路由命中率 / fallback 触发率
- 人设一致性分 / 用户满意度

**告警分级**：
- P0（电话）：服务全挂 / 隐私泄漏 / 监管事件
- P1（IM）：核心指标劣化 / 成本爆预算
- P2（日报）：长期趋势

## 七、日志与可追溯

```
每个 LLM 调用记日志：
  user_id / role_id / 模型 / prompt 版本 / 输入摘要 / 输出摘要 / 耗时 / token / 审核结果

合规要求（生成式 AI 服务管理办法）：
  - 日志保留 90 天+
  - 可追溯（哪个用户/哪个模型/什么时间生成了什么）
  - AIGC 内容标注（水印/元数据）
```

## 八、权限与隐私

```
权限（谁能做什么）：
  RBAC：用户/角色/管理员分权
  数据权限：行级过滤（用户只能看自己会话）
  工具权限：LLM 调用工具前校验权限

隐私（数据怎么保护）：
  PII 识别 + 脱敏（身份证/电话/地址不入记忆、不入 prompt）
  端到端加密（敏感会话）
  最小授权（员工后台查数据要审批 + 脱敏展示）
  数据可删除（用户行使'被遗忘权'）
  数据本地化（跨境合规）
```

## 九、预案与演练

```
预案手册：
  - 模型厂商挂了 → 切备用 + 降级
  - GPU 不够 → 限流 + 切便宜模型
  - 流量 10 倍突发 → 排队 + 拒绝低优先级
  - 隐私泄漏 → 紧急止血 + 上报监管

混沌工程：定期注入故障（杀节点/延迟/限流）演练系统韧性
```

## 十、底层本质

稳定性与安全的本质是**"给服务装上全套防护系统，把'能跑'升级为'跑得稳、跑得久、跑得安全'"**：

- 限流 = 防超载
- 降级 = 保核心
- 缓存 = 减负担
- 监控 = 早发现
- 权限隐私 = 守底线

这是把'裸奔的 AI 服务'升级为'工业级可运营系统'——是 toC AI 产品能否长期活下去的硬指标。

## 常见考点

1. **LLM 限流和传统限流区别？**——维度更多（按 token/按 QPS/按成本）、tier 差异大（VIP vs 免费）、要考虑厂商配额（自己限了不超厂商额度）、要降级（不能直接 429）。
2. **缓存会不会让角色变傻？**——会。TTL 要短、相似度阈值严、关键场景（情感支持）不缓存、定期清缓存，平衡命中率和新鲜度。
3. **隐私怎么防止被 LLM 训练进厂商模型？**——用 API（多数厂商声明不训练）/ 自部署敏感场景 / prompt 前脱敏 / 合同条款约束。


## 结构化回答

**30 秒电梯演讲：** 聊到服务稳定性与安全怎么做，我的理解是——稳定性与安全是给 AI 陪伴产品装上'刹车+保险柜'——限流降级防雪崩、缓存队列削峰、监控报警兜底、权限隐私守底线，让服务既跑得快又翻不了车。打个比方，像给一辆车装系统——限流是限速带（防超速崩）、降级是备用轮胎（爆胎还能跑）、缓存是加油站缓存油（少跑几次油站）、监控是仪表盘（随时看车况）、权限隐私是车锁和保险柜（防偷防泄）。

**展开框架：**
1. **限流** — 令牌桶/漏桶，按用户/IP/接口维度，多级（网关/应用/模型）
2. **降级** — 核心保命（强模型→弱模型→模板），非核心关闭（视频生成暂停）
3. **缓存** — prompt 结果缓存/热点角色/用户画像，相似 query 命中
