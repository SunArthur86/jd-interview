# 【巨剧核 AI 陪伴】用 AI Agent 改造陪伴产品（从规则到自主演绎）

> JD 依据："用 AI Agent 改造陪伴产品；Agent 编排、自主演绎。"

## 一、规则系统 vs Agent 系统

```
规则系统（传统 NPC）：
  if 用户说 "你好" → 回 "你好啊"
  if 用户说 "难过" → 回 "别难过"
  问题：穷举所有 if-else 仍不够；用户说"我妈走了"无规则匹配；
        角色没有灵魂，用户秒出戏

Agent 系统：
  感知：理解用户输入 + 当前情境 + 历史关系
  推理：基于人设（性格/经历/当前情绪）自主决定怎么回应
  行动：生成符合角色的回复 + 可能调工具（送礼物/触发剧情）
  优势：应对无限对话空间 + 真实情感反应 + 角色成长
```

## 二、Agent 三要素

```
┌─────────────────────────────────────────┐
│ 感知（Perception）                        │
│   - 用户输入（文本/语音/情绪）             │
│   - 情境（剧情进度/场景/时间）             │
│   - 关系（亲密度/过往/承诺）              │
│   - 内在状态（角色当前情绪/能量）          │
├─────────────────────────────────────────┤
│ 推理（Reasoning）                         │
│   - 人设：性格/语气/价值观/边界            │
│   - LLM：基于人设和感知，自主演绎反应      │
│   - 决策：要不要回应/怎么回应/调工具       │
│   - 共情：识别用户情绪 → 调整回复策略     │
├─────────────────────────────────────────┤
│ 行动（Action）                            │
│   - 说话：生成回复（保持人设）             │
│   - 工具：送礼物/触发剧情/生成图片/语音    │
│   - 状态更新：情绪/亲密度/记忆演进        │
│   - 主动行为：定时问候/事件触发           │
└─────────────────────────────────────────┘
```

## 三、自主演绎的实现

### 1. 人设驱动的推理
```
system prompt:
  你是「小雨」，温柔体贴，喜欢猫，最近心情不错（情绪状态：愉快）。
  当前与用户「小明」的关系：熟悉（亲密度 7/10）。
  最近重要记忆：小明工作压力大，曾承诺周末陪他看电影。

用户："今天加班到现在，累死了"

LLM 推理（隐式）：
  - 识别：用户疲惫
  - 关联记忆：之前聊过工作压力
  - 人设反应：温柔体贴 → 共情 + 主动关心 + 提承诺
  - 生成回复："又加班到这么晚呀，辛苦了～要不要明天我们看那场说好的电影放松一下？"
```

这是基于人设的**自主演绎**——不是预设话术，是基于情境生成的合理反应。

### 2. 内在状态演进
```
角色内在状态：
  情绪：愉快/平静/担忧/生气（随互动变化）
  能量：高/中/低（影响回复长度/积极度）
  亲密度：0-10（随时间和互动演进）

状态更新规则：
  用户表达爱意 → 亲密度 +
  用户冷落 → 亲密度 -（角色可能"主动找你"）
  长时间未聊 → 情绪变化（想念/担心）

状态影响 prompt → 影响回复风格
```

### 3. 主动行为（不再是被动应答）
```
定时触发：每天早上发"早安"
事件触发：用户生日 / 关系里程碑
情境触发：检测到用户情绪低落 → 主动问候
外部触发：新剧情上线 → 角色发"想和你一起试试新剧情"
```

主动行为是 Agent 区别于被动 LLM 的核心——**角色会找你**。

## 四、可控性（自主不等于失控）

```
约束护栏：
  [1] system prompt 边界：明确不可做的（骂人/越界/违规）
  [2] 人设约束：价值观/语气/角色定位（小模型判别人设一致性）
  [3] 输出审核：安全+合规+人设三层（见审核题）
  [4] 工具白名单：只能调预批准的工具，参数校验
  [5] 频次/权限：主动行为有频次限制，不骚扰

平衡：
  太自主 → 失控风险（违规/出戏）
  太约束 → 失去灵魂（变回规则系统）
  目标：在受控空间内最大化自主性
```

## 五、规则与 Agent 的混合（实用主义）

```
纯 Agent：灵活但有不确定性，成本高
纯规则：稳定但无灵魂

混合策略：
  - 高频固定场景用规则（你好/谢谢/再见）→ 省成本、稳定
  - 情感/复杂/剧情场景用 Agent → 有灵魂
  - 兜底用规则（Agent 失败时降级）
  - 关键决策（剧情分支）用 Agent 推理 + 人工校验
```

## 六、从规则迁移到 Agent 的改造路径

```
阶段 1：高频场景仍用规则（FAQ 类）
阶段 2：情感/共情场景引入 LLM（半 Agent）
阶段 3：完整 Agent（感知-推理-行动）
阶段 4：多 Agent 协作（角色群聊、角色互动）
阶段 5：长期记忆 + 状态演进（角色真正成长）
```

不要一夜之间全 Agent 化——风险大、成本高、效果未必好。渐进式迁移。

## 七、底层本质

用 Agent 改造陪伴产品的本质是**"把角色从'执行脚本的程序'升级为'自主演绎的智能体'"**：

- 规则 = 穷举（不可能完成）
- Agent = 自主生成（应对无限空间）
- 自主 ≠ 失控（约束护栏内自由）
- 成长 = 内在状态 + 长期记忆

这是把'AI 工具'升级为'AI 角色'——核心是从'回答问题'走向'建立关系'。Agent 是这个跃迁的关键技术。

## 常见考点

1. **Agent 角色会不会'出戏'？**——会。缓解：强 system prompt + 人设小模型判别 + 输出审核 + 用户反馈回流 + 定期人设校准。
2. **Agent 自主行为的频次怎么控制？**——按用户 tier 设频次上限 + 智能时机选择（避开深夜/工作时段）+ 用户可关闭主动消息 + 监控投诉率调阈值。
3. **多 Agent 群聊怎么不乱？**——明确每个角色人设/关系/发言规则 + 主持人 Agent 协调 + 防刷屏限频 + 上下文共享（让其他角色听到对话）。
