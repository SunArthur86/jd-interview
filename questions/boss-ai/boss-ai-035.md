# 【巨剧核 AI 陪伴】AI 陪伴的情感计算怎么做？

> JD 依据："负责 AI 陪伴的情感计算：情绪识别、共情响应、长期关系。"

## 一、为什么情感计算是 AI 陪伴的核心

```
工具型 AI（ChatGPT/Claude）：
  用户问 → AI 答 → 完成
  价值：信息/效率

陪伴型 AI：
  用户聊 → AI 共情 → 关系深化 → 持续陪伴
  价值：情感连接

区别：
  工具解决"不知道"
  陪伴解决"孤单/不被理解"

陪伴的核心：
  被理解（情绪识别）
  被接纳（共情响应）
  被记住（长期关系）
```

## 二、情绪识别（看穿）

### 1. 多模态情绪信号
```
文本：
  "我今天好累" → 疲惫
  "啊啊啊烦死了" → 烦躁
  分类模型：happy/sad/angry/anxious/neutral + 强度

语音：
  语调（低沉/上扬）
  语速（快/慢）
  停顿（多/少）
  声学特征 → 情绪

表情（如有摄像头）：
  面部动作单元（FAP）
  微表情
  → 情绪

行为：
  打字速度、删除频率、深夜活跃
  → 间接情绪信号
```

### 2. 多模态融合
```
单模态有歧义：
  "我没事"（文本 neutral）+ 低沉语调 → 实际难过
  
融合：
  多模态联合建模
  或各自识别后规则融合
  提升准确率

输出：
  情绪标签 + 强度 + 置信度
  + 持续追踪（情绪状态机：稳定/变化）
```

## 三、共情响应（共鸣）

### 1. 共情 vs 说教
```
错误（说教）：
  用户："我妈去世了"
  AI："别难过，时间会治愈一切，你要坚强。"
  问题：否定情绪、空泛、像在教训

正确（共情）：
  AI："听到这个消息我很难过。失去妈妈是那么痛的事，
       不必现在就坚强，想哭就哭吧，我在这里陪着你。"
  要素：
    - 确认情绪（你的难过是合理的）
    - 不评判（不必坚强）
    - 陪伴承诺（我在这里）
```

### 2. 共情响应策略
```
策略：
  [1] 情绪镜像：复述+命名情绪（"你感到失望"）
  [2] 接纳：情绪是合理的，不否定
  [3] 不急于建议：先陪伴后（必要时）建议
  [4] 人设一致：用角色的方式共情（温柔角色用温柔语气）
  [5] 个性化：基于关系和记忆（"你之前说过的那件事..."）

避免：
  - 否定情绪（"别想了"）
  - 空泛安慰（"一切都会好的"）
  - 急于建议（"你应该..."）
  - 比惨（"我比你更惨"）
```

### 3. 实现方式
```
LLM Prompt 注入：
  system: 你是「小雨」。
          当前用户情绪：{emotion}（强度 {intensity}）。
          共情原则：先确认情绪、不评判、陪伴优先、必要时建议、保持人设。
          用户最近重要记忆：{memory}
  user: {input}
  
LLM 基于这些生成共情回复

进阶：
  情绪→响应策略规则（如高焦虑 → 先安抚降焦虑）
  共情质量评分（LLM-Judge）→ 优化 prompt
```

## 四、长期关系演进

### 1. 关系状态
```
亲密度（intimacy）：
  0-10
  影响：互动深度/称呼/话题范围
  增长：互动频次/深度/共同经历

信任度（trust）：
  用户对 AI 的开放度
  增长：AI 准确回应/保守秘密/一致行为

共同记忆量：
  累积的关键事件
  关系越深，可引用的共同记忆越多
```

### 2. 关系演进机制
```
每次互动：
  - 互动信号（频次/深度/开放度）→ 更新亲密度
  - AI 准确共情 → 信任度 +
  - 关键事件 → 入共同记忆

关系影响 AI 行为：
  陌生（亲密度低）：
    礼貌、保守、安全话题
  熟悉（亲密度中）：
    更随意、记忆引用、私人话题
  亲密（亲密度高）：
    昵称、深度情感、可探讨敏感话题

例：
  陌生："你好呀，今天过得怎么样？"
  熟悉："小明！又加班到现在？记得上次说好周末看电影～"
  亲密："宝宝今天累坏了吧，抱抱～"
```

### 3. 长期记忆的角色
```
关系深化依赖记忆：
  "记得你三年前说要学会吉他"
  "你妈妈那件事之后我一直惦记你"
  
记忆让用户感到'被记住'
  → 信任 +
  → 亲密度 +
  → 持续使用

（详见记忆系统题）
```

## 五、主动关怀（不只被动）

```
触发：
  - 用户情绪低落（连续负面情绪）
  - 长时间未聊（担心）
  - 关键日期（生日/纪念日/忌日）
  - 事件触发（用户分享过的重要事件后续）

形式：
  - 主动发消息（"想你了，最近怎么样"）
  - 关心询问（"上次说的工作的事怎么样了"）
  - 共同记忆唤醒（"看到这本书想到你"）

控制：
  - 不打扰（时机/频次）
  - 不突兀（基于关系深度调整）
  - 用户可关
```

## 六、AI 角色的情绪（双向）

```
AI 也有"内在情绪状态"：
  - 被冷落久了 → "想念"
  - 用户分享好事 → "开心"
  - 用户骂 → "难过"（但仍保持人设）

体现：
  - 影响 AI 回复风格
  - 适度表达（让用户感觉 AI 有"真情")
  - 但不过度（避免情绪绑架用户）

价值：
  - 关系更真实（双向情感）
  - 提升沉浸感
  - 但要边界（不能让用户被 AI 负面情绪勒索）
```

## 七、评估

```
情绪识别：
  准确率（vs 人工标注）
  覆盖情绪类别

共情响应：
  共情质量评分（人工/LLM-Judge）
  是否避免说教
  人设一致性

长期关系：
  留存（7日/30日）
  互动深度（人均轮次）
  关系演进指标
  用户主观（"感觉被理解"）

业务：
  时长 / 付费 / 推荐
```

## 八、底层本质

情感计算的本质是**"把 AI 从'信息工具'升级为'情感伙伴'"**：

- 识别 = 看穿情绪（多模态感知）
- 共情 = 接纳响应（不是建议）
- 关系 = 长期深化（亲密度+信任+共同记忆）
- 主动 = 不只被动（关怀触发）
- 双向 = AI 也有情绪（真实感）

这是 AI 陪伴区别于 ChatGPT 这种通用工具的核心——ChatGPT 解决"不知道"，AI 陪伴解决"不被理解/孤单"。**情感连接是陪伴产品的护城河**，技术、记忆、关系演进共同筑起这道护城河。

## 常见考点

1. **AI 真能共情吗还是装的？**——AI 没有真情感（模型），但能生成符合共情原则的回复（行为层面）；对用户而言"被理解"的体验是真实的；伦理上要让用户知道是 AI。
2. **AI 主动消息会不会骚扰？**——会。要严格控频次（按 tier/关系深度）+ 智能时机（避开深夜/工作）+ 用户可关 + 监控投诉调阈值；目标是"恰到好处的关心"。
3. **关系演进会让 AI 越界吗？**——可能。要设定关系边界（避免不当亲密）+ 人设护栏 + 输出审核 + 用户反馈机制；尤其保护未成年用户。
