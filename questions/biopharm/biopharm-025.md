# 【生物医药 AI】AI Harness / LLMOps 全链路怎么搭？

> JD 依据："Harness、Skills、Workflow；LLM 应用开发。"

## 一、为什么需要 LLMOps

```
传统软件：逻辑确定，CI 跑测试就能保证质量
LLM 应用：
  - 概率性（同输入不同输出）
  - 会随模型升级/prompt 改/数据变而退化
  - 质量难度量（没有明确对错）
  - 会幻觉、漂移
→ 没 LLMOps 就是"黑盒上线听天由命"
```

## 二、LLMOps 闭环

```
┌─────────────────────────────────────────┐
│  网关层    统一接入 / 路由 / 限流 / 计量   │
├─────────────────────────────────────────┤
│  推理层    vLLM 高效部署 / 批处理          │
├─────────────────────────────────────────┤
│  评估层    离线评测集 + 在线质量指标        │
├─────────────────────────────────────────┤
│  监控层    性能/成本/质量/漂移 + 告警       │
├─────────────────────────────────────────┤
│  迭代层    改 prompt/模型/数据 → 回归 → 上线 │
└─────────────────────────────────────────┘
        ↑________________________|
              数据飞轮闭环
```

## 三、网关层

- 统一接入（OpenAI 兼容）、多模型路由、限流、计量（见014）。
- 全链路追踪（每次调用的模型/prompt/工具/引用）。

## 四、推理层

- 高效部署：vLLM（continuous batching）、量化、KV Cache（见026）。
- 多模型管理、自动扩缩、预热。

## 五、评估层（核心）

### 1. 离线评测集
```
从真实 query 抽样 + 人工标注标准答案 + 分场景/难度
评测指标：
  - 准确率（答案是否正确）
  - 引用命中率（是否基于检索）
  - 幻觉率
  - 完整性/相关性（LLM-as-Judge 或人工）
每次变更跑评测集，对比基线。
```

### 2. 在线质量指标
- 引用命中率、用户反馈（点踩/纠正）、质量打分、重写率。

## 六、监控层

| 监控 | 指标 | 告警 |
|------|------|------|
| 性能 | QPS/P99延迟/错误率 | 延迟/错误率突增 |
| 成本 | token/费用/缓存命中 | 成本异常增长 |
| 质量 | 引用命中/幻觉率/满意度 | 质量下降 |
| 漂移 | 输入分布/输出分布/查询类型 | 分布偏移 |

**漂移检测**：输入分布变（新类型 query）、输出分布变（质量/长度突变）、引用命中率降，都可能是退化信号。

## 七、迭代闭环（数据飞轮）

```
1. 监控发现退化 / 用户反馈 / 评测集回归失败
2. 分析根因（prompt？模型？数据过期？检索差？）
3. 改进（改 prompt / 换模型 / 更新知识库 / 优化检索）
4. 评测集回归（不能引入新问题）
5. 灰度上线 + 在线 A/B 验证
6. 全量 + 持续监控
→ 把失败 case 补进评测集，飞轮越转质量越高
```

## 八、Prompt/模型版本管理

- prompt 像代码一样版本化（Git/专门的 prompt 管理）。
- 模型版本管理（哪个模型/prompt 在哪个环境）。
- 变更触发自动回归评测，不退化才上线（AI 的 CI/CD）。
- 灰度发布（见019）控影响。

## 九、医药场景的 LLMOps 特色

- **评估要含医药专业性**：临床正确性、证据等级、合规性（不只通用指标）。
- **幻觉零容忍**：关键场景幻觉率监控 + 人工复核闭环。
- **合规审计**：每次决策可追溯（模型版本+prompt+数据版本）。
- **数据治理**：知识库更新触发评测回归（避免过时答案）。

## 十、底层本质

LLMOps 本质是**"给概率性 AI 应用装上可度量、可监控、可迭代的运维闭环"**。网关接入、推理提效、评估度量、监控发现退化、迭代优化，形成数据飞轮。

**这是 AI 应用从'上线即终点'到'持续优化'的分水岭** —— 不做 LLMOps 的 AI 应用只会越用越差，做了的才能越用越好。

## 常见考点

1. **怎么自动评估 AI 质量？**——LLM-as-Judge（用强模型评弱模型）+ 规则校验（引用命中/格式）+ 人工抽检；分场景设评测集，注意 Judge 模型本身的偏差。
2. **模型升级怎么验证？**——评测集回归（不能退化）+ 灰度 A/B（在线真实流量）+ 关注新版本新问题（不只看平均分）。
3. **怎么发现 prompt 退化？**——变更触发评测集回归 + 在线质量监控（引用命中率/满意度）+ 漂移检测，三者结合。
