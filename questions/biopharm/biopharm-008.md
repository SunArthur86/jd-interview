# 【生物医药 AI】企业知识库怎么建设（文档/版本/权限/更新）？

> JD 依据："AI 产品前后端开发：知识库、向量数据库；企业级 AI Agent 产品的架构设计、研发落地。"

## 一、知识库 ≠ 向量库

```
错误做法：把一堆 PDF 切了直接扔向量库 → 来源混乱、版本过时、权限裸奔、更新失控

正确做法：文档当资产管理 → 生命周期 / 版本 / 权限 / 元数据 / 更新同步 → 才进向量库
```

## 二、文档生命周期管理

```
[录入] → [审核] → [发布] → [使用/监控] → [更新] → [归档/下线]
```
- **录入**：上传 + 元数据必填（来源/作者/时间/类型/标签/证据等级）。
- **审核**：内容审核（准确性/合规）+ 格式校验。
- **发布**：审核通过才进向量库，对外可见。
- **监控**：被引用频率、用户反馈（有用/错误）。
- **下线**：过期/废止要召回，从向量库移除。

## 三、版本管理（医药刚需）

```
药品说明书 v1.0 (2023) → v2.0 (2024 修订)
向量库存：{chunk, doc_id, version: "2.0", status: "current"}
旧版 status: "archived"，默认不检索，可按需查
```
- 检索默认只取 `current` 版本。
- 法律/历史追溯场景可显式查旧版。
- 每个引用必须带版本（"来自说明书 v2.0 第3段"）。

## 四、权限与多租户

```
租户A ─┐
租户B ─┤→ 向量库（每个 chunk 带 tenant_id, dept, role, sensitivity）
租户C ─┘
检索时：WHERE tenant_id = ? AND dept IN (?) AND role 可见
```
- **强制隔离**：在检索层（向量库元数据预过滤）强制，不靠应用层自觉。
- **敏感分级**：公开/内部/机密，机密需额外审批才能检索。
- 详见数据治理题（015）。

## 五、增量更新（性能关键）

```
文档 v1.0 改了一页：
错误：整个文档重新切分+embedding（慢且浪费）
正确：chunk 级哈希比对 → 只重建变化 chunk → 删除旧 chunk 向量 → 插入新
```
- 每个 chunk 算 hash，对比新旧版本找变化块。
- 删除旧 chunk 的向量，插入新 chunk 的向量（带新版本号）。
- 大规模重建用异步任务 + 增量索引。

## 六、元数据治理（检索与引用的基础）

| 元数据 | 作用 |
|--------|------|
| doc_id / version | 唯一标识 + 版本追溯 |
| source / author | 来源可信度 |
| created / updated | 时效性过滤 |
| type（指南/文献/说明书） | 证据等级 |
| tags | 分类检索 |
| tenant / dept / role | 权限过滤 |

元数据越规范，过滤、排序、引用越精准。

## 七、医药知识库的特殊治理

- **证据等级标注**：RCT > 队列 > 病例 > 综述，影响结论可信度，可在检索时加权。
- **文献时效**：医学知识更新快，旧文献默认降权。
- **合规要求**：药品信息必须可追溯至官方来源（说明书/批文），不能引用非权威内容。

## 八、底层本质

企业知识库本质是**"把文档从'文件'升级为'受控资产'"**。生命周期管准入准出，版本管时效，权限管边界，增量更新管同步，元数据管治理。

**RAG 的可信度=知识库的治理成熟度** —— 没有治理的知识库，就是给 LLM 喂垃圾源头。

## 常见考点

1. **怎么保证知识库和向量库一致？**——文档变更触发增量重建任务（chunk 哈希对比），向量库写入与文档状态原子化（带 status），下线即标记/删除。
2. **大文档怎么高效更新？**——chunk 级哈希、只重建变化块、异步并行、断点续跑。
3. **怎么度量知识库质量？**——覆盖率（问题能否召回相关文档）、时效性（过期占比）、引用命中率、用户反馈准确率。


## 结构化回答

**30 秒电梯演讲：** 聊到企业知识库怎么建设（文档/版本/权限/更新），我的理解是——企业知识库不是'扔一堆 PDF 进向量库'，而是把文档当资产管起来——生命周期（录入/审批/发布/下线）、版本、权限、更新同步，才能被 RAG 可靠引用。打个比方，像药品监管——药不是随便上架的，要经过审批（录入审核）、有批号（版本）、限特定机构用（权限）、发现问题要召回（下线）。知识库里的文档同理，得有'GMP'流程。

**展开框架：**
1. **文档生命周期** — 录入→审核→发布→更新→归档→下线
2. **版本管理** — 文档多版本，检索带版本过滤（药品说明书会修订）
3. **权限** — 按租户/部门/角色控制可见性和检索范围

**收尾：** 这块我在项目里也踩过坑——想深入的话，可以接着聊：增量更新怎么做？您更想看哪个方向？
