# 【生物医药 AI】AI Infra 基础设施怎么搭（GPU/模型仓库/特征/漂移）？

> JD 依据："系统性能优化：资源调度；云原生；LLM 应用开发。"

## 一、为什么需要 AI Infra

```
无 Infra：
  各团队自己管 GPU/模型/特征 → 浪费/混乱/难演进
  模型漂移没人监控 → 上线即开始退化
  特征训练/在线不一致 → 线上效果差

有 Infra：
  GPU 共享调度、模型统一管理、特征一致、漂移预警
  → AI 应用可运行、可演进、可治理
```

## 二、AI Infra 全景

```
┌──────────────────────────────────────────┐
│  应用层    RAG/Agent/推理服务              │
├──────────────────────────────────────────┤
│  生命周期  训练→评估→部署→监控→再训练      │
├──────────────────────────────────────────┤
│  基础设施  GPU 调度 / 模型仓库 / 特征 / 监控 │
├──────────────────────────────────────────┤
│  底层      K8s / 容器 / 存储 / 网络         │
└──────────────────────────────────────────┘
```

## 三、GPU 调度（算力复用）

### 推理调度
```
[请求队列] → [continuous batching] → [GPU 池]
  ↑优先级（实时>批量）
  ↑多模型分时复用（显存够时）
  ↑冷热模型换入换出
  ↑自动扩缩（按负载）
```
- vLLM 等推理引擎内部有批处理调度（见026）。
- 多租户公平性：配额+优先级。

### 训练调度
```
训练任务 → 任务队列 → GPU 集群
  ↑多卡并行（数据/张量/流水线并行）
  ↑抢占式（高优先级抢低优先级）
  ↑断点恢复（checkpoint）
  ↑弹性（ Spot 实例回收容忍）
```

## 四、模型仓库（版本管理）

```
模型仓库管：
  - 版本（v1/v2/...）
  - 元数据（架构/参数/训练数据版本/超参）
  - 评测结果（哪些指标/评测集）
  - 血缘（哪个数据/代码训的）
  - 部署环境（哪些环境跑了哪个版本）
  - 制品（权重文件）

→ 像 Git 管代码一样管模型
→ 可对比/回滚/复现
```
- 工具：MLflow / DVC / 自研。
- 配合 CI/CD（评测通过才部署，见025）。

## 五、特征管理（数据一致）

### 问题
```
训练用：特征 = log(用量) （离线批算）
在线用：特征 = 用量 （忘了 log）
→ 训练-在线不一致，线上效果差（训练-在线 skew）
```

### 特征存储（Feature Store）
```
统一特征定义 → 一次开发
  - 离线：训练读历史特征（批）
  - 在线：服务读实时特征（低延迟）
  - 一致：同一逻辑，离线在线结果一致
- 特征共享（多模型复用）
- 特征版本/血缘
工具：Feast / Tecton / 自研
```

## 六、漂移监控（退化预警）

```
上线后模型会退化，原因：
  - 数据漂移：输入分布变（新类型 query/新药）
  - 模型漂移：预测分布变
  - 概念漂移：输入-输出关系变（如指南更新）

检测：
  - 数据漂移：KS检验/PSI/分布距离
  - 模型漂移：预测分布/置信度变化
  - 业务指标：准确率/点击率/满意度降

→ 超阈值告警 → 触发再训练/微调
```

### 闭环
```
监控漂移 → 收集新数据 → 再训练/微调 → 评估 → 灰度上线 → 监控
→ 模型越用越好，而非越用越差
```

## 七、全生命周期编排

```
[数据准备] → [特征工程] → [训练] → [评估] → [部署] → [监控]
     ↑________________________________________________|
                    漂移触发再训练
```
- 每步可追溯（数据/代码/模型版本）。
- 自动化 pipeline（CI/CD for ML）。
- 工具：Kubeflow / Airflow / MLflow / 自研。

## 八、医药场景的 Infra 特色

- **GPU 调度**：医药数据敏感，可能专用 GPU 集群（合规隔离）。
- **模型仓库**：辅助诊断模型属医疗器械，版本/数据/评测全程可追溯（合规审计）。
- **特征管理**：医学特征（检验值/基因）标准化、单位一致（医疗安全）。
- **漂移监控**：医学知识更新快（指南修订/新药），漂移监测+再训练更关键。

## 九、底层本质

AI Infra 本质是**"把 AI 全生命周期的公共能力沉淀成可复用的基础设施"**。GPU 调度管算力，模型仓库管版本，特征管理管一致，漂移监控管退化，形成闭环。

**AI Infra 是'AI 能用起来'到'AI 能规模化、可持续演进'的底座** —— 没有它，每个团队重复造轮子，模型上线即退化。

## 常见考点

1. **训练-在线 skew 怎么避免？**——特征存储统一离线在线逻辑，一次开发两边用；代码 review 确保一致；上线前做 skew 检测（离线在线特征对比）。
2. **模型漂移怎么发现？**——监控输入分布（数据漂移）、预测分布（模型漂移）、业务指标（准确率/满意度），用 KS/PSI 等统计检验，超阈值告警。
3. **GPU 怎么高效利用？**——推理用 continuous batching + 多模型分时复用 + 冷热换入换出；训练用多卡并行 + 任务队列 + 抢占；配合弹性扩缩。
