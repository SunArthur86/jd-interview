# 【生物医药 AI】可观测性/治理/跨团队推进怎么做？

> JD 依据："4 人+团队管理经验；跨团队协作（产品/算法/前端/数据）；高可靠。"

## 一、为什么 AI 特别需要可观测性

```
传统服务：逻辑确定，出问题看日志能定位
AI 服务：
  - LLM 决策是黑盒（为什么这么答？）
  - 概率性（同输入不同输出）
  - 多步链路（意图→检索→工具→生成→校验）
  - 会幻觉（事实错误）
→ 不可观测 = 不可治理、不可信任、出事定位不了
```

## 二、三支柱（看系统健康）

### 1. 指标（Metrics）—— 看状态
```
技术：QPS / P99延迟 / 错误率 / GPU利用率 / 队列堆积
业务：任务成功率 / token消耗 / 成本 / 缓存命中
质量：引用命中率 / 幻觉率 / 用户反馈
```
- Prometheus + Grafana 仪表盘。

### 2. 日志（Logs）—— 看细节
- 结构化日志（JSON），含 request_id 串联。
- 关键事件：模型调用/工具调用/校验失败/人工审批。

### 3. 链路追踪（Traces）—— 看全程
```
一次请求的 span：
  [意图分类 200ms] → [检索 500ms] → [工具调用 800ms]
  → [生成 2s] → [校验 300ms] → [返回]
```
- OpenTelemetry + 自定义 span，每段记输入输出/模型版本/token。
- 定位"哪一步慢/哪一步错"。

## 三、SLO/SLI（量化可用性）

```
SLI（指标）：如 请求成功率、P99 延迟 < 5s、质量得分 > 4.0
SLO（目标）：如 成功率 99.9%、P99 < 5s
错误预算：1 - SLO（如 0.1%）
  → 预算用完，停新功能专抓稳定性
```
- 用数字而非感觉定"够不够可用"。
- 按业务重要性分级（核心高 SLO）。

## 四、AI 决策可追溯（合规与信任）

```
每次 AI 决策记录：
{
  request_id, user, tenant,
  input_query,
  model_version, prompt_template_version,
  tools_called: [...], tool_results: [...],
  retrieved_chunks: [{doc_id, version, 段落}],
  generated_answer, citations,
  verifier_report,
  human_approved_by, timestamp
}
```
- 用户反馈"答错了" → 追溯当时用了哪个模型、检索了什么、引用了什么。
- 合规审计：医药决策必须可追溯（见数据治理题015）。

## 五、质量与幻觉监控

```
在线质量监控：
  - 引用命中率（生成是否基于检索内容）
  - 幻觉率（抽样校验事实 vs 检索证据）
  - 用户反馈（点踩率、纠正率）
  - 质量得分（小模型/规则打分）
异常 → 告警 → 排查（模型变差？知识过期？prompt 问题？）
```

## 六、跨团队推进（工程文化 + 流程）

```
4人+团队管理 + 跨团队协作（产品/算法/前端/数据）：

技术层面：
  - 清晰接口契约（API/OpenAPI）
  - 数据驱动决策（用指标/SLO 说话）
  - 文档化架构和决策（ADR）

流程层面：
  - 需求评审（产品提需求，工程评估可行性）
  - 算法与工程协同（算法出模型，工程出可靠性）
  - 定期对齐（周会/里程碑）

团队层面：
  - 明确分工与责任人
  - Code Review + 技术分享
  - 培养 AI 工程文化（实验/度量/迭代）
```

## 七、底层本质

可观测性和治理本质是**"把 AI 黑盒变白盒，让状态可看见、性能可度量、决策可追溯、质量可信任"**。三支柱看健康，SLO 量化目标，决策留痕保可信，质量监控防幻觉，工程文化推进协作。

**这是 AI 系统从'能跑'到'可治理、可信任、可协作'的分水岭** —— 不可观测的 AI 永远是 demo，可观测的 AI 才是企业产品。

## 常见考点

1. **怎么定位 AI 响应慢？**——链路追踪看各 span 耗时（意图/检索/工具/生成/校验），找最慢一段；常见瓶颈是检索（向量库）和生成（LLM）。
2. **怎么监控 AI 质量？**——在线（引用命中率/幻觉率/用户反馈）+ 离线评测集定期回归；建立质量基线，异常告警。
3. **跨团队怎么推 AI 项目落地？**——清晰接口+数据驱动+里程碑+利益对齐；工程提供可靠性底座，算法提供能力，产品定义价值，用 demo 和指标推进共识。
