# 【生物医药 AI】AI Agent 可观测性怎么建（trace/token/审计/幻觉）？

> JD 依据："高可靠；可观测性；LLM 应用开发。"

## 一、为什么 Agent 特别需要可观测性

```
传统服务：逻辑确定，看日志能定位
Agent：
  - 多步链路（意图→检索→工具→生成→校验）
  - 每步概率性（LLM 决策不确定）
  - 会幻觉（编内容）
  - 成本高（token 贵）
  - 影响业务（医药影响健康）
→ 不可观测 = 出事定位不了、无法信任、无法优化
```

## 二、可观测四要素

### 1. Trace（链路追踪）

```
一次 Agent 调用的 span 树：
[agent_run 3.8s]
  ├─[classify_intent 0.2s] {model, prompt_v3, input_tokens:50}
  ├─[retrieve 0.5s] {kb:药品库, top5: [...], 命中:是}
  ├─[tool:check_interaction 0.8s] {args, result, status}
  ├─[generate 2.0s] {model:gpt4, input:1.2k, output:300}
  └─[verify 0.3s] {citation_hit:是, hallucination:低}
```
- 每步一个 span，记输入输出/模型/prompt版本/token/耗时。
- 定位"哪步慢、哪步错、哪步贵"。
- 工具：OpenTelemetry / LangSmith / Arize Phoenix。

### 2. Token 监控（成本）

```
按 [步骤][模型][租户][场景] 记录：
  - 输入 token / 输出 token
  - 耗时 / 成本
  - 重试次数 / fallback

→ 发现：检索结果太长导致输入 token 爆？某模型贵？
→ 优化：压缩检索结果 / 换小模型（见010成本控制）
```

### 3. 决策审计（合规与追溯）

```
每次 Agent 决策留痕：
{
  request_id, user, tenant, timestamp,
  input_query,
  model_version, prompt_template_version,
  steps: [
    {type:"intent", output:"用药咨询"},
    {type:"retrieve", chunks:[{doc_id, version, 段落}]},
    {type:"tool", name:"check_interaction", result:...},
    {type:"generate", answer, citations},
    {type:"verify", report}
  ],
  human_approved_by: "张医生"
}
```
- 用户反馈"答错了" → 追溯当时用了什么模型/检索了什么/引用了什么。
- 医药合规审计必备（见015数据治理）。

### 4. 幻觉检测（质量）

```
在线检测：
  - 引用命中率：生成内容是否基于检索片段（字面/语义比对）
  - 事实校验：关键事实 vs 检索证据（小模型/规则）
  - 用户反馈：点踩/纠正率
  - 抽样人工：定期抽样人工核查

统计幻觉率，超阈值告警。
```

## 三、指标与告警

| 维度 | 指标 | 告警 |
|------|------|------|
| 性能 | 端到端P99/各步耗时 | 延迟突增 |
| 成本 | token/费用/重试 | 成本异常 |
| 质量 | 引用命中/幻觉率/满意度 | 质量下降 |
| 业务 | 任务成功率/工具准确率 | 成功率降 |
| 漂移 | 输入分布/输出分布 | 分布偏移 |

## 四、医药 Agent 的特殊观测

- **引用完整性**：每个结论可溯源到文档版本（合规）。
- **幻觉零容忍**：关键场景（用药/诊断）幻觉率监控 + 人工复核闭环。
- **决策可解释**：记录推理链（为什么这么建议），便于专家审查。
- **审计合规**：决策留痕满足监管（见015）。

## 五、可观测驱动的优化闭环

```
监控发现异常（如某类问题幻觉率高）
   ↓
trace 定位（是检索召回差？还是生成跑偏？）
   ↓
分析根因（知识库缺内容？prompt 不好？模型不行？）
   ↓
改进（补知识/改prompt/换模型）
   ↓
评测回归 + 上线验证
   ↓
持续监控
```

## 六、底层本质

Agent 可观测性本质是**"把多步概率决策的黑盒变透明，可调试、可信任、可优化、可追溯"**。Trace 追每步决策，token 监控成本，决策审计留痕，幻觉检测保质量。

**这是 Agent 从'能跑'到'可治理'的分水岭** —— 不可观测的 Agent 永远是黑盒 demo，可观测的才能成为可信的企业产品。

## 常见考点

1. **怎么定位 Agent 答错？**——trace 看每步 span：意图分错？检索召回差？工具结果错？生成跑偏？校验漏过？逐段定位根因，不只看最终答案。
2. **幻觉怎么自动发现？**——引用命中校验（生成 vs 检索比对）+ 事实校验（关键事实 vs 证据）+ 用户反馈 + 抽样人工，多手段结合统计幻觉率。
3. **token 监控有什么用？**——发现成本瓶颈（哪步最贵/哪个模型费），驱动优化（压缩检索/换小模型/缓存），按租户计量计费。
