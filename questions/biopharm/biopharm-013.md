# 【生物医药 AI】Workflow + Agent 编排引擎架构怎么设计？

> JD 依据："Agent、RAG、Workflow、Tool Calling 核心能力建设；负责企业级 AI Agent 产品的架构设计。"

## 一、为什么需要融合编排

```
纯 Workflow（如审批流）：可靠但死板，每个分支都要预定义
纯 Agent（LLM 自主规划）：灵活但不可控，可能乱跑/死循环

真实业务（如药物综述）：
  骨架要确定（检索→筛选→总结→审批→发布）
  局部要灵活（检索词怎么扩展、筛选标准怎么判）
→ 融合：Workflow 定骨架，Agent 填局部
```

## 二、引擎核心抽象

```
任务图（DAG）：节点 + 边（依赖/条件/循环）
状态机：每节点状态（PENDING/RUNNING/PAUSED/SUCCESS/FAILED）
调度器：按依赖+状态选下一批可执行节点
执行器：执行具体节点（LLM/工具/人工/子流程）
检查点：每步状态+输入输出持久化（断点恢复）
```

## 三、四类节点

| 节点类型 | 作用 | 示例 |
|----------|------|------|
| LLM 节点 | 调模型生成/分类/抽取 | 意图分类、答案生成 |
| 工具节点 | 调外部工具/API | 查药品库、检索文献 |
| 人工节点 | 暂停等审批 | 用药建议确认、报告签发 |
| 子流程节点 | 嵌套另一个 Workflow | 复用通用子流程 |

## 四、流程定义（示例）

```yaml
workflow: drug_literature_review
nodes:
  - id: classify_intent, type: llm
  - id: search, type: tool, parallel: [pubmed, internal_kb]
  - id: rerank, type: llm
  - id: summarize, type: llm
  - id: approve, type: human       # 合规审批
  - id: publish, type: tool
edges:
  - classify_intent -> search
  - search -> rerank -> summarize -> approve -> publish
  - approve.rejected -> rerank      # 驳回重做
```
- 支持并行（search 的两路同时跑）、条件、循环（驳回回退）。
- 用 YAML/可视化拖拽定义，不写死代码。

## 五、调度与执行

```
调度器循环：
  1. 找所有 PENDING 且依赖满足的节点
  2. 提交执行器（并发上限控制）
  3. 执行完更新状态 + 写检查点
  4. 人工节点：PAUSE + 通知 + 等回调
  5. 失败：重试/补偿/转人工
  6. 全部完成 → 汇总结果
```

## 六、断点恢复（长任务生命线）

```
进程挂了 → 重启 → 读检查点 → 从最近 SUCCESS 节点续跑
（节点要幂等：重试不会重复副作用，如带幂等键）
```

## 七、循环与防死循环

```
循环：驳回回退、查不到重查
防死循环：
  - max_iterations（流程级/节点级）
  - 超时熔断
  - 重复状态检测（相同输入跑 N 次仍失败 → 转人工）
```

## 八、可观测与治理

- **节点级追踪**：每节点输入/输出/耗时/状态/重试次数。
- **流程级视图**：可视化当前跑到哪、卡在哪。
- **版本化**：流程定义版本，支持灰度发布和回滚。
- **审计**：人工审批记录、关键决策留痕（合规）。

## 九、自研 vs 框架

| 选项 | 适用 |
|------|------|
| LangGraph | AI 图式编排，节点是 LLM/工具，适合 Agent 重场景 |
| Temporal | 强一致长事务，适合企业级长流程+补偿 |
| Airflow | 数据 pipeline，偏批处理 |
| 自研 | 强定制（医药合规/审计/特殊节点），有研发能力 |

**实战**：强合规+长流程可用 Temporal（骨架）+ LangGraph（局部 Agent）；强定制自研。

## 十、底层本质

编排引擎本质是**"把复杂业务流程抽象为可执行、可恢复、可观测的任务图"**。Workflow 提供确定性骨架，Agent 提供局部灵活性，状态机保证可恢复，检查点保证不丢。

**这是 AI 从'单次调用'到'企业级流程'的关键基础设施** —— 没有可靠编排，Agent 永远只是 demo。

## 常见考点

1. **怎么平衡 Workflow 的确定性和 Agent 的灵活性？**——骨架（关键步骤/审批/合规）用 Workflow 确定性编排，局部（开放检索/表达）用 Agent 灵活决策；边界按"是否需确定性"划分。
2. **长流程怎么不丢不重？**——检查点持久化 + 节点幂等（幂等键/状态判断）+ 重启续跑。
3. **引擎性能瓶颈在哪？**——节点调度开销、状态库写入、检查点 IO；用批量/异步/缓存优化，热点流程预热。
