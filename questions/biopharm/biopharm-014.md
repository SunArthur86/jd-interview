# 【生物医药 AI】多模型路由与推理平台架构怎么设计？

> JD 依据："AI 产品前后端开发：多模型路由、缓存；系统性能优化：并发、成本控制。"

## 一、为什么要推理平台

```
业务直连各模型：
  业务A → OpenAI SDK
  业务B → 通义 SDK
  业务C → 文心 SDK
  → 各家 API 不同、无统一限流计量、OpenAI 挂了业务停、成本失控、切换难

推理平台：
  业务 → 统一网关 → 路由到多模型 → 统一计量监控
```

## 二、三层架构

```
┌──────────────────────────────────────────┐
│  网关层    路由 / 限流 / 鉴权 / 计量 / 流式 │
│         （OpenAI 兼容协议统一入口）        │
├──────────────────────────────────────────┤
│  推理层    自部署（vLLM）+ 外部 API 供应商   │
│         （批处理 / KV Cache / 量化）       │
├──────────────────────────────────────────┤
│  可观测层  监控 / 成本 / 质量 / 日志 / trace │
└──────────────────────────────────────────┘
```

## 三、网关层（统一入口）

### 1. 统一协议（屏蔽异构）
```
业务统一调：POST /v1/chat/completions  （OpenAI 兼容）
网关后端适配：OpenAI / 通义 / 文心 / Claude / 自部署
```
- 上层只调一套 API，换模型不改业务代码。
- 开源网关：LiteLLM / one-api / 自研。

### 2. 路由（核心）
```python
def route(req):
    scene = classify(req)                 # 场景
    complexity = estimate(req)            # 复杂度
    # 场景+复杂度+成本+延迟选模型档位
    model = pick_model(scene, complexity, budget)
    return model_with_fallback_chain(model)
```

### 3. 限流与计量
- 按租户/模型限流（令牌桶）。
- 记 token 用量与费用，按租户出账。

### 4. 流式透传
- SSE 流式透传到业务，首字延迟低。

## 四、推理层

### 1. 自部署推理优化
- **vLLM**：continuous batching + PagedAttention，吞吐数倍。
- **KV Cache**：复用公共前缀（如系统 prompt）。
- **量化**：INT8/INT4 降显存换少量精度。
- 详见 LLM 推理优化题（026）。

### 2. 多模型 GPU 夢用
```
一张 GPU 跑多模型（显存够时）：分时复用 / 冷热模型换入换出
或：多模型各占 GPU，按负载动态扩缩
```

### 3. 外部 API 供应商
- 接多家（OpenAI/通义/文心/Claude）做 fallback 和成本优化。

## 五、fallback 与容灾

```
请求 → 主模型（Claude）
        ├ 超时/限流/5xx → 备（GPT）
        ├ 仍失败 → 自部署开源模型
        └ 都不行 → 规则兜底 + 告警
```
- 网关层健康检查 + 自动切换，对业务透明。
- 多供应商 + 自部署混合，单点故障不阻断。

## 六、可观测层

```
按 [租户][模型][场景] 统计：
  - QPS / P99 延迟 / 错误率 / 限流命中
  - token 消耗 / 费用 / 缓存命中
  - 质量指标（用户反馈/评测得分）
```
- 质量-成本曲线驱动路由策略优化。
- 成本预算告警，超限降级。

## 七、自部署 vs API 动态混合

```
稳定基线流量 → 自部署（单 token 成本低）
突发流量 → API 弹性（按量，不浪费固定资源）
任务类型：
  实时交互 → 自部署（低延迟）
  批处理 → API（省运维）或离线自部署
平台按负载和成本动态调度。
```

## 八、底层本质

推理平台本质是**"把多模型异构性屏蔽，统一成可路由、可观测、可降级、成本可控的服务"**。网关统一入口，OpenAI 兼容协议屏蔽差异，路由找最优匹配，批处理提效，多供应商容灾。

**这是'能用 AI'到'规模化、可控地用 AI'的基础设施** —— 没有推理平台，企业 AI 成本和可用性都不可控。

## 常见考点

1. **网关怎么不影响延迟？**——路由判定 ms 级（轻量分类/规则），与请求可并行；流式透传不加 buffer；网关本身异步非阻塞。
2. **怎么选自部署模型？**——按任务难度选规模（7B/14B/70B），按显存选量化，离线评测对齐业务质量基线；成本 = GPU折旧/吞吐。
3. **多供应商怎么保证一致体验？**——统一协议 + 评测各供应商质量基线 + 路由时考虑差异（弱模型仅用于简单场景）。
