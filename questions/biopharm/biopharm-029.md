# 【生物医药 AI】医药知识图谱怎么构建（实体/关系/GraphRAG）？

> JD 依据："金融/医药垂直 AI 经验优先；RAG、向量数据库；医药 AI。"

## 一、为什么医药需要知识图谱

```
纯文本/向量检索的局限：
  "药物A通过什么靶点治疾病B？有什么替代药？"
  → 需多跳推理（药→靶点→疾病→其他药）
  → 向量检索难表达结构化关系
  → 知识图谱：实体-关系网，支持精确查询和多跳

医药知识高度关联：
  药物 - 靶点 - 疾病 - 基因 - 通路 - 不良反应
  → 天然适合图谱建模
```

## 二、图谱构建流程

```
[数据源]                    [抽取]                [图谱]
文献(PubMed/指南)  ─┐
说明书/药典        ─┤→ 实体抽取 → 关系抽取 → 三元组 → 融合去重 → 图谱
结构化DB(DrugBank) ─┤   (NER)     (RE)     (头,关,尾)  + 人工校验
临床/不良反应库    ─┘
```

### 1. 实体抽取（NER）
```
文本 → 识别实体：
  药物（阿司匹林）、疾病（心梗）、靶点（COX-1）、基因、通路、不良反应
方法：
  - 小模型 NER（BERT 系，快）
  - LLM few-shot（灵活，复杂实体）
```

### 2. 关系抽取（RE）
```
"阿司匹林抑制COX-1" → (阿司匹林, 抑制, COX-1)
"阿司匹林用于心梗预防" → (阿司匹林, 预防, 心梗)
方法：
  - 关系抽取模型（分类头）
  - LLM few-shot 抽三元组
  - 远程监督（用已知三元组回标语料）
```

### 3. 融合与去重
- 不同来源同一实体对齐（"阿司匹林"="Aspirin"="ASA"）。
- 关系冲突解决（按证据等级/来源可信度）。

### 4. 人工校验
- 抽取结果抽样人工校验，医药知识错误代价大。

## 三、关系建模（医药特色）

```
节点类型：药物/靶点/疾病/基因/通路/不良反应/适应症
边类型：
  治疗（药→病）
  靶点（药→靶点）
  相互作用（药→药）
  引发（药→不良反应）
  关联（基因→病）
  ... 

边带属性：
  证据等级（RCT/队列/病例）
  来源（哪篇文献/DB）
  版本（知识时效）
  置信度
```
**证据等级和来源是医药图谱区别于通用图谱的关键**（影响可信度）。

## 四、GraphRAG（图谱+向量融合）

```
用户问："药物A的替代治疗？"

[1] 实体识别：识别 query 中的"药物A"和"替代治疗"
[2] 图谱查询：
    找药物A→治疗的疾病→其他治疗该疾病的药（多跳）
[3] 向量检索：检索相关文献/指南
[4] 融合：图谱子图 + 文档片段 一起喂 LLM
[5] 生成 + 引用回溯（图谱关系+文献）
```

### Microsoft GraphRAG 思路
- 把图谱按社区聚类，每个社区生成摘要。
- 全局问题（"总结某领域"）用社区摘要答，局部问题用图谱查询答。

## 五、图谱存储与查询

```
图数据库：Neo4j / NebulaGraph / JanusGraph
查询语言：Cypher（Neo4j）/ nGQL
查询：
  MATCH (d:Drug {name:"阿司匹林"})-[: treats]->(dis: Disease)
  RETURN dis  // 查阿司匹林治什么病

多跳：
  MATCH (d)-[:treats]->(dis)<-[:treats]-(alt:Drug)
  WHERE d.name="阿司匹林" RETURN alt  // 替代药
```

## 六、质量与治理

- **证据等级**：RCT > 队列 > 病例，影响关系可信度。
- **来源标注**：每个关系溯源到文献/DB。
- **时效更新**：医学知识更新快，增量抽取 + 过期标注。
- **人工校验**：医药错误代价大，关键关系必校验。

## 七、底层本质

医药知识图谱本质是**"把海量关联的医药知识结构化为实体-关系网，支持精确查询和多跳推理"**。实体抽取找节点，关系建模连边，GraphRAG 融合向量+图谱提升复杂问答。

**医药图谱的可信度=证据等级+来源溯源+人工校验** —— 这区别于通用百科图谱，是医药 AI 落地的知识基石。

## 常见考点

1. **GraphRAG 比纯向量 RAG 强在哪？**——结构化关系查询（多跳：药→靶点→病→其他药）、精确（图谱关系确定）、可解释（路径可展示）；纯向量靠语义相似，复杂关系和多跳弱。
2. **怎么保证图谱质量？**——多源交叉验证 + 证据等级 + 来源溯源 + 抽样人工校验 + 增量更新；医药错误代价大，校验不可或缺。
3. **图谱和向量库怎么选？**——结构化关系/多跳推理用图谱，非结构化文本检索用向量；GraphRAG 融合两者，按场景权重。


## 结构化回答

**30 秒电梯演讲：** 聊到医药知识图谱怎么构建（实体/关系/GraphRAG），我的理解是——医药知识图谱是把'分散在文献/数据库里的医药知识'结构化成'实体+关系'的网——实体抽取找节点（药/靶点/疾病），关系建模连边（治疗/相互作用），GraphRAG 让检索能多跳推理。打个比方，像给医药知识画一张地铁图——站点是实体（药物/疾病/基因），线路是关系（治疗/引发/相互作用），查'从A到B怎么走'就是多跳推理。LLM 直接读文本是看文字描述，有图谱能直接查'网'，答得更准。

3. **图谱构建** — 文献抽取+结构化数据库融合+人工校验
