# 【拼多多供应链】设计采购协同平台（SRM）

> JD 依据："供应商管理"。

## 一、SRM 全流程

```
寻源 → 供应商准入 → 合同签订 → 采购订单 → 到货 → 质检 → 对账 → 付款
 │       │            │          │        │      │      │      │
寻源单  供应商档案   框架协议    PO      到货单  质检单  账单   付款单
```

## 二、供应商管理

```sql
CREATE TABLE supplier (
    id BIGINT,
    name VARCHAR,
    level ENUM('A','B','C','D'),   -- 分级
    score DECIMAL,                 -- 综合评分
    categories JSON,               -- 供货类目
    status ENUM('合作','暂停','淘汰')
);
```

**分级标准**：
- A：交期准时率 > 98%、质量合格率 > 99%、价格竞争力强
- D：淘汰（多次质量问题）

## 三、自动补货

```java
public void autoReplenish(Long skuId) {
    long stock = stockService.get(skuId);
    long safety = configService.getSafetyStock(skuId);
    if (stock < safety) {
        long forecast = forecastService.predict(skuId, 30);  // 30 天预测销量
        long leadTime = supplierService.getLeadTime(skuId);  // 采购提前期
        long orderQty = forecast * (leadTime + 7) / 30 - stock;  // 补到提前期+7天
        purchaseOrderService.create(skuId, orderQty);
    }
}
```

## 四、协同门户

供应商在线：
- 接收采购单（确认/拒绝）
- 录入发货信息
- 查看对账单
- 提交发票

降低沟通成本，全流程留痕。

## 五、合同管理

- **框架协议**：长期价格约定（季度谈价）
- **执行单**：每次采购引用框架协议价格
- **价格变更**：需审批，版本化

## 六、底层本质

SRM 本质是**"采购流程的数字化协同"**——把线下采购搬到线上，用系统保证透明、高效、可追溯。核心价值是降低采购成本（比价、谈判、自动化）和提升协同效率（门户、自动）。

## 常见考点
1. **怎么选供应商**？——RFQ（报价请求）+ 综合评分（价格/质量/交期/服务）。
2. **自动补货的难点**？——销量预测准确度（大促/季节性），提前期波动。
3. **采购和财务怎么对接**？——三单匹配（采购单/到货单/发票）+ 自动对账。

## 苏格拉底式面试追问

> 这组追问不是背答案，而是模拟面试官层层逼近本质。每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：SRM 的自动补货你用"预测销量 × (提前期+7天) - 当前库存"算补货量。但大促时预测销量不准（可能低估 10 倍），自动补货反而备货不足。为什么不直接人工补货？**

自动补货的价值是"常态化效率"，人工补货的价值是"异常判断"。两者不矛盾，分场景：
1. **日常补货（自动）**——非大促期间，销量相对稳定（历史均值 + 季节性），预测模型准确（误差 < 20%），自动补货高效（省人力、及时）。
2. **大促补货（人工 + 预测辅助）**——大促销量暴涨且波动大，预测模型可能失准（历史数据没有同类大促参考）。这时自动补货只做"建议量"，采购经理结合"大促目标、营销策略、供应商产能"人工决策。
所以策略是"日常自动、大促人工审核"，自动补货的阈值（如补货量 > 10000）触发人工审批，避免模型失准导致过度采购或缺货。

### 第二层：证据与定位

**Q：供应商反馈"采购单没收到"，你怎么定位是 SRM 发送失败、供应商门户 bug，还是供应商账号问题？**

三段排查：
1. **看 SRM 发送日志**——`purchase_order_log` 表查该采购单的发送记录（发送时间、渠道、状态）。如果是"已发送"但供应商没收到，是渠道问题（邮件进垃圾箱、站内信未读）。
2. **看供应商门户**——用供应商账号登录门户，查"待处理采购单"，如果列表为空但 SRM 显示"已发送"，是门户的数据同步问题（如门户读的是从库，有延迟）。
3. **看供应商状态**——`SELECT status FROM supplier WHERE id=?`，如果 status='PAUSED'（暂停合作），采购单可能被过滤（不发给暂停供应商）。查供应商的账号状态（是否禁用、密码过期）。

### 第三层：根因深挖

**Q：自动补货生成的采购单金额异常高（比历史均值高 50 倍），差点下单。根因是什么？**

根因大概率是"预测模型输入异常"：
1. **历史数据脏数据**——某天的销量数据异常（如测试数据误写入，某 SKU 单日销量 10 万），预测模型把这个异常点当真实销量，算出超高补货量。查 `sales_history` 表的异常值（`WHERE qty > avg * 10`）。
2. **大促参数误配**——大促系数（如"双 11 销量是日常 10 倍"）配错了（设成 100 倍），补货量乘以 100。查 `promo_factor` 配置。
3. **提前期异常**——供应商提前期数据错（如本应 7 天配成 70 天），`(leadTime + 7)` 算出超大补货量。查 `supplier_lead_time` 配置。
根因定位：打印 `autoReplenish` 的中间变量（`forecast`、`leadTime`、`orderQty`），看哪个值异常。预防：补货量设上限（如单次 < 10 万），超限触发人工审批。

**Q：那为什么不直接把自动补货关了，全部人工，彻底避免模型失准导致的过度采购？**

全人工补货的代价是"人力成本 + 时效差"：
1. **人力不够**——拼多多千万 SKU，每天有几千个 SKU 触发补货点，全人工 review 不现实（要几十个采购员）。
2. **时效慢**——人工 review 要几小时，热销 SKU 几小时就可能缺货。自动补货秒级触发，时效好。
3. **一致性**——人工判断主观（不同采购员对同一 SKU 的补货量不同），自动化用统一规则，一致性高。
所以策略是"自动为主 + 人工兜底异常"——常规补货自动（量级合理），异常补货（量级超阈值）人工审批。不是二选一。

### 第四层：方案权衡

**Q：供应商分级你用 A/B/C/D 四级（基于交期、质量、价格）。但某 A 级供应商最近交期延迟（延迟率从 2% 涨到 10%），系统还是按 A 级分配采购份额。怎么动态调整？**

分级不能是静态的（年度评一次），要动态：
1. **滚动评分**——每周/每月重算供应商评分（`交期准时率 × 0.4 + 质量合格率 × 0.4 + 价格竞争力 × 0.2`），评分下降自动降级。某 A 级供应商延迟率涨到 10%，准时率从 98% 降到 90%，评分下降，可能降到 B 级。
2. **份额联动**——采购份额分配按分级（A 级拿 50%、B 级 30%、C 级 20%），降级后自动减少份额，转给其他 A 级供应商。
3. **预警**——评分下降超阈值（如一个月内降 10 分）触发飞书告警采购经理，人工介入（是否暂停合作、约谈）。
动态分级让供应商有压力持续改进，避免"一旦评 A 就躺平"。

**Q：为什么不直接淘汰交期不准的供应商，换新的？**

换供应商成本高：
1. **准入成本**——新供应商要审核资质、试订单验证质量、谈价格，周期几周到几个月。
2. **磨合成本**——新供应商初期交期/质量不稳定（不熟悉流程），可能比老供应商更差。
3. **议价权**——老供应商合作久有信任和议价空间（量大折扣），新供应商没有。
所以策略是"动态分级 + 份额调节"——表现差降级减份额（保留合作但减少依赖），持续差才淘汰。给改进机会，而不是一差就换。只有 D 级（多次严重质量问题）才淘汰。

### 第五层：验证与沉淀

**Q：你怎么证明 SRM 的自动补货真的降低了缺货率、没有过度采购？**

两个核心指标：
1. **缺货率**——`oos_count`（out of stock，库存为 0 的 SKU 数 / 总 SKU 数），上线自动补货后应从 5% 降到 < 1%。分品类看（热销品缺货率更低）。
2. **库存周转率**——`inventory_turnover = COGS / avg_inventory`，自动补货应让周转率提升（备货精准不积压）。如果周转率下降，说明过度采购（库存堆积）。
3. **补货准确率**——`forecast_error = |实际销量 - 预测销量| / 实际销量`，应 < 20%；> 50% 说明预测模型要优化（加特征、调参数）。

**Q：怎么让 SRM 的采购流程透明可追溯，避免暗箱操作？**

三件事：
1. **全流程留痕**——所有采购操作（寻源/报价/选供应商/下单/审批）记录 audit log（操作人 + 时间 + 变更前后值），不可篡改，审计可查。
2. **比价透明**——同一采购需求的多个供应商报价系统化记录，选供应商时显示"为什么选 A 不选 B"（评分明细），防止人为偏向。
3. **权限分离**——寻源人（找供应商）、采购员（下单）、审批人（审金额）角色分离，单人不能完成全流程（防腐败）。大额采购（> 100 万）双人审批 + 总监复核。
