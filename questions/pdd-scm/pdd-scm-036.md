# 【拼多多供应链】怎么用 AI 做销量预测和智能补货？

> JD 依据："大模型/机器学习探索"。

## 一、销量预测模型

**特征工程**：
```python
features = {
    "历史销量": [近 7/14/30 天销量, 同比, 环比],
    "季节性": [月份, 节假日, 换季标记],
    "大促": [双11/618 标记, 历史大促销量],
    "商品属性": [类目, 价格, 评价数],
    "外部": [天气, 竞品价格, 热点事件]
}
```

**模型选择**：
| 模型 | 适用 |
|------|------|
| Prophet | 强季节性、节假日 |
| LSTM | 复杂非线性时序 |
| XGBoost | 多特征表格 |
| Transformer（时序） | 长序列、多变量 |

**输出**：
```json
{
  "sku": 12345,
  "forecast_30d": 2000,
  "confidence_interval": [1800, 2200],
  "mape": 0.15  // 误差 15%
}
```

## 二、智能补货算法

```python
def auto_replenish(sku):
    forecast = predict(sku, days=30)          # 未来 30 天预测
    lead_time = supplier_lead_time(sku)        # 采购提前期
    safety_stock = forecast.std() * 1.65       # 安全库存（95% 服务水平）
    current = stock_service.get(sku)

    order_qty = forecast.mean() * (lead_time + 7) / 30  # 提前期+7 天
                 - current + safety_stock
    if order_qty > 0:
        create_purchase_order(sku, order_qty)
```

## 三、难点与解法

**1. 大促预测（历史样本少）**：
- 相似大促迁移学习（去年双 11 → 今年双 11）
- 实时调整（大促当天根据前几小时销量动态修正）

**2. 新品冷启动（无历史）**：
- 相似商品类比（同类目相似属性商品的销量曲线）
- 专家初始预测 + 早期销量快速修正

**3. 长尾商品（销量稀疏）**：
- 聚类（相似长尾合并预测）
- 降低补货频率（按需补货而非定期）

## 四、和 Agent 结合

Agent 调用预测 Tool 做综合决策：
```
Agent: 调预测 Tool（销量 2000）+ 供应商 Tool（A 最优）+ 库存 Tool（现 500）
     → 决策：向 A 补 2000，附预测和评估理由
```

## 五、底层本质

销量预测本质是**"用历史规律+多维信号预测未来"**：
- 库存成本（积压资金）和缺货成本（失销售）的权衡
- 预测越准，权衡越优
- AI 让预测从"经验估计"到"数据驱动"

**经济价值**：库存周转率提升 10% → 释放大量现金流（拼多多千亿级库存）。

## 常见考点
1. **预测准确率怎么评估**？——MAPE（平均绝对百分比误差）、WAPE（加权）、库存周转率（业务）。
2. **大促预测难点**？——历史样本少 + 突发性，用迁移学习 + 实时修正。
3. **预测和补货的关系**？——预测是输入，补货是决策（还需考虑提前期/安全库存/供应商）。

## 苏格拉底式面试追问

> 这组追问不是背答案，而是模拟面试官层层逼近本质。每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：销量预测你用 AI 模型（XGBoost/Transformer），但传统方法（移动平均、指数平滑）简单可解释。为什么要用复杂模型？**

复杂模型能捕捉"非线性 + 多变量"：
1. **非线性**——销量受价格/促销/季节/竞品影响，关系不是线性的（如降价 10% 销量涨 50%，不是线性）。移动平均只看历史均值，捕捉不到这种非线性。XGBoost/Transformer 能学复杂特征交互。
2. **多变量**——销量受"价格 + 类目趋势 + 天气 + 节假日 + 大促"多因素影响，移动平均只看销量本身。机器学习模型能融合多维特征（如"降价 + 双 11"组合预测大涨）。
3. **大促/新品**——传统方法依赖稳定历史，大促（历史样本少）和新品（无历史）预测差。AI 模型用迁移学习（相似商品/相似大促）能预测。
代价是"可解释性差"（AI 模型是黑盒，预测出 2000，说不清为什么）。简单商品（销量稳定）仍用移动平均够用，复杂场景（大促/新品/多变量）才用 AI。分层使用。

### 第二层：证据与定位

**Q：预测模型上线后，某 SKU 预测 2000，实际只卖了 500（误差 300%）。你怎么定位是模型问题、特征输入错，还是外部突发事件？**

三步排查：
1. **看特征输入**——预测时的特征值（价格/促销/历史销量）是否正确。如果"历史销量"特征取了脏数据（如某天测试数据误写 10 万），模型被误导。查特征工程的 pipeline。
2. **看模型决策**——用 SHAP/LIME 等可解释性工具，看模型预测 2000 的"贡献特征"。如果"大促系数"特征贡献最大（如大促系数=5），但实际没大促，是大促特征配错（误标记该 SKU 参加大促）。
3. **看外部事件**——如果特征都对（价格/促销/历史），但实际销量低，可能是外部事件（如竞品降价/负面新闻/供应链断货）。查这段时间的运营事件日志。
定位：对比"预测时模型看到的特征"和"实际情况"，找偏差源。

### 第三层：根因深挖

**Q：定位发现某类目（服饰）的预测系统性偏高（MAPE 40%，其他类目 15%）。根因是什么？**

服饰类目预测难有行业共性根因：
1. **季节性强**——服饰受季节影响大（夏装 5-8 月卖，冬装 10-1 月卖），模型如果没学好季节特征（或训练数据跨年不够），预测偏差大。解法是加"季节特征"（月份/节气）或用 Prophet（专门处理季节性）。
2. **潮流波动**——服饰受潮流影响（某款突然爆红或过气），历史规律失效。解法是加"社交媒体热度"特征（如某 SKU 的搜索量/讨论量），捕捉潮流信号。
3. **SKU 多生命周期短**——服饰 SKU 多（颜色/尺码组合），单个 SKU 历史短（一季就下架），样本稀疏。解法是按 SPU 或类目聚合预测（不按单 SKU），或用相似 SKU 迁移学习。
根因定位：看服饰类目的特征重要度（是否缺季节/潮流特征）+ SKU 历史长度（是否样本太少），针对性补特征或换策略。

**Q：那为什么不直接放弃预测服饰类目，全部人工补货（运营凭经验）？**

放弃 AI 预测回到人工会丢规模效应：
1. **规模**——拼多多服饰类目千万 SKU，人工补货要几千个运营，成本高。
2. **一致性**——人工补货主观（不同运营判断不同），AI 模型统一逻辑，一致性高。
3. **AI 仍有价值**——即使 MAPE 40%，AI 预测比"纯随机"或"历史均值"好（提供基线），人工在基线上微调（如运营知道某款要爆，手动加量）。AI + 人工优于纯人工。
所以策略是"AI 出基线 + 人工修正"，不是放弃 AI。重点优化服饰类目的特征（季节/潮流），把 MAPE 从 40% 降到 25%，配合人工修正足够可用。

### 第四层：方案权衡

**Q：大促预测（双 11）历史样本少（只有去年一次），模型预测不准。你用迁移学习（去年双 11 → 今年）。但今年和去年差异大（如市场环境变），迁移可能失准。怎么权衡？**

大促预测要"迁移 + 实时修正"：
1. **迁移学习做基线**——用去年双 11 的销量曲线（加今年日常销量的比例调整）做初始预测。如去年双 11 卖 1 万，今年日常是去年日常的 1.5 倍，预测今年双 11 = 1 万 × 1.5 = 1.5 万。这是基线（不准但有参考）。
2. **预售数据修正**——双 11 前有预售期（定金/加购），预售数据是实时的强信号。预售量 × 转化率（如 60%）修正预测。如预售 8000，预测最终 8000/0.6 = 1.3 万。
3. **实时滚动预测**——双 11 当天，根据前几小时销量动态修正（如前 2 小时卖了 3000，全天预测 3000 ×（全天/前 2 小时比例））。越接近实际越准。
策略是"事前迁移基线 + 预售修正 + 事中滚动修正"，分阶段提升准确率，不是一次预测定终身。

**Q：为什么不直接多备货（预测不准就备 2 倍），彻底避免缺货？**

多备货的代价是"库存积压"：
1. **资金占用**——备 2 倍库存占 2 倍采购资金，供应链千亿级库存，多备 10% 就是百亿资金占用，财务成本高。
2. **仓储成本**——多余库存占仓储空间，仓储费 + 管理成本。
3. **滞销风险**——没卖完的库存要清仓（打折/毁损），直接亏损。服饰类目季末清仓损失尤其大（过季衣服贬值）。
所以补货是"缺货成本 vs 积压成本"的权衡，预测越准权衡越优。盲目多备货（2 倍）虽然不缺货，但积压成本远超缺货损失。最优补货量是 `预测 + 安全库存（按预测误差和服务水平算）`，不是拍脑袋翻倍。

### 第五层：验证与沉淀

**Q：你怎么证明 AI 预测比传统方法/人工预测更准、业务价值更大？**

对比验证：
1. **准确率对比**——同样 SKU，AI 预测 vs 移动平均 vs 人工预测，对比 MAPE（平均绝对百分比误差）。AI 应比移动平均低 20%+（MAPE 从 30% 降到 15%），比人工低 10%（AI 更稳定）。
2. **业务指标**——AI 补货后的"缺货率"（库存为 0 的 SKU 比例）应下降（如从 5% 降到 2%），"库存周转率"应提升（如从 30 天降到 25 天）。这两个指标直接关联财务价值。
3. **A/B 测试**——10% SKU 用 AI 预测补货，90% 用原方法，对比一段时间后的缺货率和周转率。AI 组明显优于对照组才全量推广。
4. **持续监控**——预测准确率（MAPE）按类目/时间监控，异常下降（如某类目 MAPE 突涨）触发模型 retrain 或特征 review。

**Q：怎么让预测模型可持续优化（不一次性上线就不管）？**

闭环机制：
1. **自动 retrain**——模型每周/每月用最新销量数据 retrain（滚动更新），适应市场变化（如疫情后消费习惯变了）。retrain 后跑测试集验证，准确率不退化才上线。
2. **特征工程迭代**——定期评估新特征（如"社交媒体热度""天气""竞品价格"），加入模型看是否提升准确率。拼多多预测模型有几百个特征，持续筛选有效的。
3. **Bad case 分析**——每周分析预测误差最大的 Top 100 SKU，找共性（如某类目/某时间段系统性偏差），针对性加特征或调模型。
4. **模型版本管理**——MLflow 管理模型版本，每个版本的准确率/特征/训练数据记录，可回滚（新版本退化时切回旧版本）。
