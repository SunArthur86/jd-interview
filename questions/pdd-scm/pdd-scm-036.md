# 【拼多多供应链】怎么用 AI 做销量预测和智能补货？

> JD 依据："大模型/机器学习探索"。

## 一、销量预测模型

**特征工程**：
```python
features = {
    "历史销量": [近 7/14/30 天销量, 同比, 环比],
    "季节性": [月份, 节假日, 换季标记],
    "大促": [双11/618 标记, 历史大促销量],
    "商品属性": [类目, 价格, 评价数],
    "外部": [天气, 竞品价格, 热点事件]
}
```

**模型选择**：
| 模型 | 适用 |
|------|------|
| Prophet | 强季节性、节假日 |
| LSTM | 复杂非线性时序 |
| XGBoost | 多特征表格 |
| Transformer（时序） | 长序列、多变量 |

**输出**：
```json
{
  "sku": 12345,
  "forecast_30d": 2000,
  "confidence_interval": [1800, 2200],
  "mape": 0.15  // 误差 15%
}
```

## 二、智能补货算法

```python
def auto_replenish(sku):
    forecast = predict(sku, days=30)          # 未来 30 天预测
    lead_time = supplier_lead_time(sku)        # 采购提前期
    safety_stock = forecast.std() * 1.65       # 安全库存（95% 服务水平）
    current = stock_service.get(sku)

    order_qty = forecast.mean() * (lead_time + 7) / 30  # 提前期+7 天
                 - current + safety_stock
    if order_qty > 0:
        create_purchase_order(sku, order_qty)
```

## 三、难点与解法

**1. 大促预测（历史样本少）**：
- 相似大促迁移学习（去年双 11 → 今年双 11）
- 实时调整（大促当天根据前几小时销量动态修正）

**2. 新品冷启动（无历史）**：
- 相似商品类比（同类目相似属性商品的销量曲线）
- 专家初始预测 + 早期销量快速修正

**3. 长尾商品（销量稀疏）**：
- 聚类（相似长尾合并预测）
- 降低补货频率（按需补货而非定期）

## 四、和 Agent 结合

Agent 调用预测 Tool 做综合决策：
```
Agent: 调预测 Tool（销量 2000）+ 供应商 Tool（A 最优）+ 库存 Tool（现 500）
     → 决策：向 A 补 2000，附预测和评估理由
```

## 五、底层本质

销量预测本质是**"用历史规律+多维信号预测未来"**：
- 库存成本（积压资金）和缺货成本（失销售）的权衡
- 预测越准，权衡越优
- AI 让预测从"经验估计"到"数据驱动"

**经济价值**：库存周转率提升 10% → 释放大量现金流（拼多多千亿级库存）。

## 常见考点
1. **预测准确率怎么评估**？——MAPE（平均绝对百分比误差）、WAPE（加权）、库存周转率（业务）。
2. **大促预测难点**？——历史样本少 + 突发性，用迁移学习 + 实时修正。
3. **预测和补货的关系**？——预测是输入，补货是决策（还需考虑提前期/安全库存/供应商）。
