# 【拼多多供应链】可观测性、稳定性治理、跨团队推进怎么做？

> JD 依据："积极推动落地解决"（跨团队推进能力）。

## 一、可观测性三支柱

```
Metrics（聚合指标）   → 看趋势、告警（Prometheus）
Logs（事件日志）       → 看细节（ELK）
Traces（调用链路）     → 定位环节（SkyWalking）

协同: 告警(Metrics) → 定位环节(Trace) → 看细节(Log)
```

**供应链关键指标**：
```
业务: order_qps, order_p99_rt, 库存准确率, 对账差异率
系统: jvm_gc, db_slow_query, redis_hit_rate
```

## 二、稳定性治理（SLO）

```
SLI（指标）: P99 RT、可用性、错误率
SLO（目标）: P99 < 200ms、可用性 99.99%
错误预算 = 1 - SLO = 0.01%（月停 4.3 分钟）
```

**错误预算管理**：
- 预算未用完 → 可大胆发布（创新）
- 预算用完 → 停止发布（稳定优先）

## 三、跨团队推进（钱学森"协调运转"）

**1. 接口契约**：
```yaml
# 契约文件（OpenAPI/Protobuf），双方共同维护
openapi: 3.0
paths:
  /inventory/deduct:
    post:
      requestBody: { skuId: long, qty: int }
      responses: { 200: { success: bool } }
```
契约先行，避免"你改了我不知道"。

**2. 联合演练**：
- 大促前联合演练（商品+库存+订单+结算）
- 模拟故障（杀实例）验证降级

**3. 变更协同**：
```
变更日历: 标注各团队发布计划
封网期: 大促/节假日前禁止非紧急变更
联合发布: 跨服务依赖的变更协同上线
```

**4. 事故复盘（ blameless ）**：
- 不追责个人，找系统漏洞
- 5Why 根因分析
- 改进措施跟踪到闭环

## 四、供应链治理实战

**库存准确率 SLO**：
```
SLI: 库存 Redis vs DB 差异率
SLO: 差异率 < 0.01%
告警: 超 0.01% 自动告警 + 对账
```

**跨团队接口（商品→订单）**：
```
商品团队改 SKU 模型 → 提前 2 周通知订单团队
→ 双方约定迁移方案 → 灰度上线
```

## 五、底层本质

可观测性 + 治理 + 跨团队是钱学森**"能协调运转"**原则的落地：
- **看得清**（可观测性）：系统状态透明
- **管得住**（治理）：可用性量化管理
- **协同好**（跨团队）：多团队有序协作

这是大规模工程系统的管理基础——技术再好，管理跟不上也会出问题。

## 常见考点
1. **SLO 和 SLA 区别**？——SLO 是内部目标（团队承诺），SLA 是外部合同（违约赔偿）。
2. **错误预算怎么用**？——预算内可激进发布（创新），超预算保守（稳定）。
3. **跨团队最大难点**？——信任和沟通，解法是契约先行 + 联合演练 + blameless 复盘。
