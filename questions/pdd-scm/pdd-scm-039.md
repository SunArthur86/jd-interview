# 【拼多多供应链】FDE 怎么落地一个供应链 AI 项目？

> JD 依据：FDE 是 AI 时代复合角色。

## 一、FDE 全链路

```
客户需求（降库存成本）
   ↓ FDE 拆解
业务目标（库存周转率 +10%）
   ↓ FDE 设计
技术方案（AI 预测 + 智能补货 + 异常监控）
   ↓ FDE 推进
落地（数据准备→模型训练→集成→灰度）
   ↓ FDE 负责
效果（周转率实际提升 12%）
```

## 二、案例：智能补货项目

**需求**：某品类库存积压严重，缺货也频繁。

**FDE 方案**：
```
数据层: 整合历史销量 + 季节 + 大促 + 供应商提前期
AI 层: 销量预测（XGBoost）+ 智能补货（规则+预测）
监控层: 预测准确率 + 库存周转 + 缺货率
运营层: 控制台配置规则 + A/B 实验
```

**落地路径**：
- Phase 1（1 月）: 数据整合 + 基线预测
- Phase 2（2 月）: 补货算法 + 灰度
- Phase 3（持续）: 自学习迭代

**效果**：
- 库存周转率 +15%
- 缺货率 -30%
- 年节约库存成本 2000 万

## 三、FDE 能力模型

- 业务理解（30%）：供应链全流程、库存成本结构
- 技术深度（30%）：AI 全栈、分布式系统
- 沟通（20%）：客户+技术+跨团队
- 项目管理（20%）：需求/进度/风险

## 四、底层本质

FDE 本质是**"业务-技术-客户的翻译官"**：
- 业务语言（降成本）↔ 技术语言（预测模型）↔ 客户语言（ROI）

这是 AI 时代的"系统化思维"——不只看技术或业务，全链路负责。

## 常见考点
1. **FDE 和架构师区别**？——FDE 更贴近客户负责落地效果；架构师偏技术方案。
2. **AI 项目最大挑战**？——业务-技术脱节，FDE 桥接。
3. **怎么衡量 FDE 价值**？——交付率、客户 NPS、业务 ROI。
