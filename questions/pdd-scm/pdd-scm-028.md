# 【拼多多供应链】负载均衡、隔离、压测怎么做？

> JD 依据："高可用、高并发环境下的研发"。

## 一、负载均衡

```
DNS → VIP（LVS）→ Gateway → 服务实例
```

**算法**：
- 轮询、随机（简单）
- 最少连接（处理时长不均）
- 一致性哈希（缓存场景）
- 同机房优先（减少跨机房 RT）

## 二、隔离

```
物理级（最强）：不同业务不同集群
集群级：K8s namespace
线程级：每下游独立线程池
信号量级：计数隔离
```

**供应链隔离**：
```java
@Bean("corePool") Executor corePool() { ... }   // 下单/库存（核心）
@Bean("auxPool") Executor auxPool() { ... }      // 推荐/评价（非核心）
```

## 三、全链路压测

```
1. 录制线上流量
2. 构造影子流量（× N 倍）
3. 影子库隔离（不污染生产）
4. 全链路监控
5. 找瓶颈 → 扩容 → 验证
```

**容量规划**：
```
日常 5万 QPS，双 11 峰值 100万
冗余 1.5 倍 → 设计容量 150万
单机 5000 QPS → 300 实例
```

## 四、底层本质

三者是分布式系统运维基础：
- 均衡：效率最大化（避免热点）
- 隔离：稳定性保障（爆炸半径）
- 压测：容量可知（数据驱动）

## 常见考点
1. **一致性哈希**？——节点增减只影响相邻段（缓存场景减少 miss）。
2. **线程池隔离 vs 信号量**？——线程池强（独立资源）开销大；信号量轻共享线程。
3. **压测怎么不打扰用户**？——影子流量 + 影子库 + 影子 MQ。
