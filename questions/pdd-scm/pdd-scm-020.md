# 【拼多多供应链】采购到结算的全流程怎么设计？

> JD 依据："大供应链体系包括商品、供应商、仓配、运营和结算"。

## 一、正向全流程

```
采购 → 到货 → 质检 → 入库 → 上架 → 销售 → 出库 → 对账 → 结算
 │       │      │      │      │      │      │      │      │
采购单  到货单  质检单  库存+   SKU    订单   运单   账单   付款
                     现货    上架                 对账   凭证
```

## 二、各环节聚合根 + 状态机

**采购单**：
```
新建 → 已审批 → 已下单 → 部分到货 → 全部到货 → 已关闭
```

**到货单**：
```
待到货 → 已到货 → 质检中 → 质检通过 → 入库中 → 已入库
                  质检失败 → 退货
```

**库存**：
```
在途（采购未到货）→ 现货（可售）→ 冻结（下单未发货）→ 已出库
```

**结算**：
```
待对账 → 对账中 → 对账一致 → 待付款 → 已付款 → 已关账
                 对账差异 → 人工核查
```

## 三、领域事件串联

```
采购到货 → ArrivalEvent → 仓储服务消费 → 质检入库
质检通过 → QcPassedEvent → 库存服务消费 → 加现货
下单 → OrderCreatedEvent → 库存服务消费 → 冻结库存
出库 → ShippedEvent → 结算服务消费 → 生成账单
```

**落地**（本地消息表 + MQ）：
```java
@Transactional
public void arrival(ArrivalCmd cmd) {
    arrivalDao.insert(arrival);                    // 业务
    outboxDao.insert(new Outbox("ArrivalEvent"));  // 事件（同事务）
}
```

## 四、对账系统

**T+1 全量对账**：
```
库存系统数据 vs 仓储系统数据 vs 财务系统数据
  → 三方一致：平账
  → 三方差异：告警 + 人工核查
```

**实时差异监控**：库存负数、金额对不上等实时告警。

## 五、逆向流程（退货/退款）

```
用户退货 → 退货单 → 质检 → 退货入库（加现货）→ 退款 → 反向结算
```

逆向是正向状态机的镜像，复用质检/入库/结算能力。

## 六、底层本质

供应链本质是**"价值流的信息映射"**——商品实物流动（采购→仓储→配送）伴随信息流（单据状态）和资金流（对账结算）三流合一。

**DDD 的价值**：每个环节是独立限界上下文（采购域、仓储域、结算域），通过领域事件松耦合，让长流程可演进、可扩展。

**和传统 ERP 区别**：电商供应链是"事件驱动 + 最终一致"（高并发），ERP 是"过程驱动 + 强一致"（内部管理）。

## 常见考点
1. **状态机怎么实现**？——状态枚举 + 状态转移表（合法性校验）+ Spring StateMachine 框架。
2. **对账怎么保证准确**？——T+1 全量三方对账 + 实时差异监控 + 差异自动调账规则。
3. **供应链核心难点**？——多部门多系统协作（一致性）+ 海量数据（性能）+ 复杂业务规则（正确性）。
