# 【拼多多交易】用户画像系统怎么设计？

> JD 依据："用户平台研发"。

## 一、标签体系

```
基础标签: 年龄/性别/地区/注册时长
行为标签: 购买频次/客单价/活跃时段/浏览品类
偏好标签: 偏好类目/价格区间/品牌偏好
时序标签: 最近活跃/购买周期/流失风险
```

## 二、计算架构

```
离线（Spark T+1）: 全量历史算长期标签
实时（Flink 毫秒）: 当前行为算短期标签（如近 1 小时浏览）
存储: HBase（全量亿用户）+ Redis（活跃用户热缓存）
服务: 推荐/营销/风控调用，RT < 20ms
```

## 三、推荐应用

```java
public List<Product> recommend(long uid) {
    UserProfile profile = profileService.get(uid);  // 拉画像
    // 用偏好类目 + 价格区间 + 协同过滤召回
    return recallByProfile(profile);
}
```

## 四、底层本质

画像是"把行为数据压缩成可用特征"——支持个性化（推荐/营销）和智能化（风控/预测）。

## 常见考点
1. **实时和离线怎么一致**？——同 SQL 双跑 + 对账。
2. **冷启动**？——默认标签 + 相似用户 + 快速学习。
3. **隐私合规**？——脱敏存储 + 授权采集 + 可关闭。
